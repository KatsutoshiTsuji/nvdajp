1. Introducción

1.1. Un Vistazo Rápido

NVDA es un lector de pantalla para el sistema operativo Windows, que le permite al usuario acceder al computador sin la necesidad de visión alguna. Consultando al sistema operativo dónde se encuentra el usuario, qué ventanas y controles están en la pantalla y observando de cerca los cambios del sistema operativo. El NVDA puede procesar y hablar a través del sintetizador de voz esta información, de manera que el usuario entienda qué está sucediendo exactamente en la pantalla.

NVDA viene con un sintetizador integrado llamado ESpeak, y también soporta tanto sintetizadores SAPI4 como SAPI5.

NVDA se basa en el uso de Microsoft active accesibility (msaa), llamadas a las API de Windows de bajo nivel y modelos de objetos de aplicaciones específicas. NVDA no usa controladores de vídeo o conectadores con la pantalla..la ventaja de todo esto es que el NVDA puede trabajar con diferentes configuraciones de sistema y no depende de una configuración particular de color o de modo de pantalla de vídeo. NVDA tampoco tiene que instalar ningún driver especial o archivo que requiera reiniciar el PC antes de ejecutar el programa por primera vez.

NVDA no soporta aún dispositivos braille, aunque estamos trabajando actualmente en esto.

NVDA puede mostrar su interface de usuario, y hablar sus mensajes, en once idiomas diferentes.

NVDA debería ser capaz de leer texto en cualquier idioma, siempre y cuando tengas un sintetiza

NVDA es un programa experimental y está bajo desarrollo; también es de código abierto; esto significa que cualquiera puede ver el código fuente o cambiarlo a su gusto para satisfacer sus necesidades . Debido a esto puede haber errores y las cosas pueden no funcionar como se espera en todos los sistemas.

Pero según más y más personas prueben el NVDA y provean de retroalimentación al usuario, el NVDA debe ser capaz de crecer e incluso mejorar, actuando como una solución completamente gratuita para usuarios de Windows que sean ciegos o deficientes visuales.

NVDA trabaja con muchos programas y solo depende de cuan accesibles sean éstos, para el buen funcionamiento del NVDA con ellos.

A parte de las funcionalidades generales de Windows, el NVDA funciona con Word pad, block de notas,outlook expres e internet explorer. También tiene soporte básico para Microsoft Word 2000/xp/2003 y Microsoft excel 2000/xp/2003. Aún no ha sido probado con Microsoft Office 2007.

NVDA también tiene extensiones para Mozilla fire fox 2.0 y superiores , siendo el explorador recomendado para obtener mayor accesibilidad con el NVDA.

io no es de ningún modo una guía completa para usar el NVDA y Windows pero si la información necesaria para que comiences a usar el NVDA, navegar y aprender por ti mismo como hacer muchas otras cosas.

2. Requerimientos del Sistema

NVDA corre en Windows xp y Windows vista, también puede correr en Windows 2000 pero no demasiadas personas han probado esto aún.

NVDA no funcionará en Windows 95/98/me, ni trabaja aún en versiones de Windows de 64 bits.

Deberías tener como mínimo 256 MB de memoria ram; el NVDA correrá mejor en PC que posean 1.0 ghz de velocidad o superiores. Aunque el NVDA correrá en máquinas mas lentas, su velocidad de respuesta y actividades mas intensas como la carga de páginas web por ejemplo, pueden darse muy lentamente

3. Obteniendo NVDA

Si aún no tienes una copia del programa, puedes descargarla visitando el sitio web en la siguiente dirección : http://www.nvda-project.org/

Ve a la sección de descargas, allí encontrarás un link para bajarte la última revisión del programa.

Hay actualmente dos modos en los que el NVDA está empaquetado. Uno es un instalador que te hablará a lo largo de la instalación de NVDA así que puedes ejecutarlo entonces en cualquier ocasión desde el Menú de Inicio. La otra es un fichero zip que contiene una versión portátil completa de NVDA, la cual puedes ejecutar desde cualquier sitio, incluyendo un lápiz de memoria USB o un CD.

4. Instalando NVDA

4.1. El Instalador de NVDA

Si tienes el instalador de NVDA, sencillamente presiona intro o haz doble clic sobre el fichero y el instalador se iniciará.

El instalador primero te hablará con alguna voz pregrabada, y te pregunta si el resto de la instalación debería ser hablada o no. Si dicdes que sí, una copia temporal de NVDA arrancará, la cual te permitirá completar la instalación con asistencia de voz.

Simplemente sigue todas las indicaciones, y NVDA debería ser instalado perfectamente para usarlo.

Al final de la instalación, el Instalador iniciará automáticamente la nueva copia de NVDA que ha sido instalada.

4.2. El Fichero zip Portátil

Si tienes la versión del fichero zip de NVDA, entonces todo lo que tienes que hacer es descomprimir este fichero en cualquier lugar donde quieras en tu disco duro. Podrías también descomprimirlo a un medio portátil tal como un lápiz de memoria USB o un CD.

5. Iniciando NVDA

Si has instalado el NVDA con el Instalador, entonces arrancar NVDA es tan fácil como presionar control+alt+n, o choosing NVDA desde el menú NVDA en el menú Programas de tu Menú Inicio.

Para iniciar la versión portátil en fichero zip, ve al directorio donde hayas descomprimido NVDA, y presiona intro o haz doble clic sobre nvda.exe.

Cuando el NVDA arranque, escucharás en primer lugar una serie de sonidos ascendentes que te indican que el programa se está cargando. entonces debes escuchar que el NVDA verbaliza “ NVDA activado". . dependiendo de cuan rápido sea tu ordenador o de si se está ejecutando el programa desde una memoria USB o algún otro medio mas lento, podría tomar unos instantes para iniciar. Si se está tomando un tiempo extra mas largo, el NVDA debe verbalizar “cargando subsistema, aguarda por favor”.

Si no escuchas nada de esto, o escuchas el sonido de error de Windows, o una serie de sonidos descendentes, significa que el NVDA presenta un error; es probable que necesites comunicar el error a los desarrolladores. Por favor revisa la página Web de NVDA para saber cómo hacer esto.

Una vez que el NVDA arranque exitosamente, debe traer al frente su ventana de interface, que te permitirá controlar las configuraciones del programa, y salir del mismo.

Muchas teclas de órdenes son mencionadas a lo largo del resto de esta guía del usuario, pero una manera fácil de explorar las diferentes teclas de órdenes es activar la ayuda de teclado.

Para activar la ayuda de teclado, presiona insert+1. Para desactivarla de nuevo, presiona insert+1 otra vez. Mientras se está en la ayuda de teclado, presionando cualquier tecla se anunciará qué hace (si se diseñó que haga algo). Las teclas no llevarán a cabo su función en este momento mientras se esté en el modo de ayuda de teclado, así puedes presionar las teclas que quieras. Por favor note que la mayoría de las teclas de órdenes de NVDA son una tecla en combinación con la tecla insert. Sin embargo, las teclas del teclado numérico llevan también a cabo por sí solas acciones especiales.

Otra tecla de órdenes útil a tener en cuenta aquí es insert+f12, la cual anuncia la hora y la fecha actuales.

6. La Interface del Usuario

El NVDA posee una ventana de interface que contiene una barra de menús, con menús que te permitirán hacer cambios en la configuración del programa y salir del mismo. También tiene una guía rápida con una lista con la mayoría de las órdenes de teclado que puedes usar mientras estés ejecutando el NVDA.

La ventana de la interface del NVDA es un buen lugar para acostumbrarse a navegar con el programa, dado que tiene un documento por donde te puedes desplazar con las flechas, además de los muchos menús y cuadros de diálogo.

6.1. mostrando y ocultando la Interface del Usuario

Aunque la ventana de la interface del NVDA aparece cuando inicias el programa, no tienes que estar todo el tiempo navegando por ella. No hay nada peor que tener demasiadas aplicaciones abiertas y tratar de encontrar una en particular a través del uso de alt+ tab.

Para esconder la ventana de interface del NVDA solo debes presionar la combinación alt+f4 estando dentro de la misma ventana; esto no te hará salir del NVDA, solo esconderá la ventana de la interface.

Para mostrar la ventana en cualquier momento, sólo debes presionar la combinación insert+n, o dar enter o doble clic sobre el icono en la bandeja de sistema de Windows.

6.2. guardando y recuperando la configuración

El NVDA guarda sus configuraciones en un archivo llamado NVDA.ini, probablemente nunca tengas que tocar dicho archivo, pero para usuarios avanzados, este archivo les permite cambiar algunas configuraciones extras que aún no tienen los menús o cuadros de texto para hacerlo.

El NVDA no guarda las configuraciones automáticamente, así que si sales del programa y lo reinicias, las mismas volverán a los valores originales. para guardar los cambios debes escoger la opción guardar del menú NVDA presionando enter sobre la misma, o en su defecto presionar las teclas control+s estando en la ventana de Interface del NVDA.

Si el NVDA está en un sistema de solo lectura como un CD por ejemplo, no estará disponible la opción de guardar la configuración y éste te dirá eso.

Si cometiste un error con la configuración y necesitas regresar a la configuración guardada, puedes hacerlo seleccionando la opción Volver a la configuración guardada" ubicada en el menú NVDA, o puedes pulsar las teclas control+r estando en la ventana del NVDA.

6.3. El Menú Preferencias

En la barra de menú de la interface del usuario, Hay un menú Preferencias que contiene algunos elementos de menú que abren cuadros de diálogo para controlar varias opciones del NVDA.

Puedes controlar las opciones de la interface del usuario, elección de sintetizador, opciones de voz, eco del teclado, opciones del ratón, presentación de objetos, opciones del modo virtual, y opciones de formateado de texto.

7. Navegando con NVDA

7.1. Navegando con el foco

La manera más común de navegar por Windows con el NVDA es simplemente moviéndose con los comandos normales del teclado, como por ejemplo tab o shift+tab para adelantar o retroceder entre objetos, presionando Alt para acceder a la barra de menús y luego navegar por ellos con las flechas, usando Alt+tab. para moverse entre aplicaciones en proceso; haciendo esto, el NVDA anunciará lo que tiene el foco.

Cuando el NVDA anuncia un objeto (Ej. cuando recibe el foco), usualmente dice el nombre del objeto , tipo, valor,descripción, atajo de teclado e información de su posición. Esto es bastante similar en otros lectores de pantalla, sin embargo los objetos pueden ser algo diferentes.

Hay algunas teclas de órdenes útiles cuando nos movemos con el foco:

  • insert+Flecha Arriba - anuncia el foco actual
  • insert+b - lee la pantalla entera en primer plano (útil para leer un cuadro de diálogo)
  • insert+t - Anuncia el título de la actual ventana en primer plano

7.2. Navegación de Objetos

Si deseas navegar alrededor de algunos objetos pero sin mover el foco, puedes usar la navegación por objetos del NVDA, esto te permitirá moverte objeto porobjeto, como en una estructura de árbol.

La estructura en árbol de los objetos, comienza en el escritorio de Windows; el tronco del árbol. las ramas principales del árbol son las aplicaciones actualmente en proceso; y por supuesto éstas aplicaciones tienen sus propias ramas (los objetos) representando varias partes de la aplicación.

La estructura puede volverse bastante larga pero eso te permite navegar en un orden lógico ; así podrás encontrar rapidamente la información que necesitas.

Esta estructura no tiene nada que ver con la forma en que los objetos se colocan en la pantalla , pero si con la forma como lógicamente se relacionan. puede tomar un poco de tiempo acostumbrarse a esto, pero una vez que lo logres, debes estar preparado para moverte fácilmente.

Actualmente el NVDA no puede navegar los objetos por el diseño de una pantalla debido a que no usa controlador de video ni conectadores de pantalla. El NVDA escoje no interactuar con la pantalla, mucho de esto porque sobre todo debe permitir el despeje y una percepción más lógica de una aplicación y del sistema operativo, en lugar de tratar de obtener una imagen plana de la pantalla.

No todas las aplicaciones tienen una estructura en árbol perfecta, pero esto depende del desarrollador de la aplicación en particular; muchas parecen estar bien, ciertamente muchos cuadros de diálogo y aplicaciones altamente estructuradas parecen tenerlos correctos.

Para navegar por objetos, utiliza los siguientes comandos:

  • Verbalizar objeto actual: insert+ 5 del teclado numérico
  • Movérse al objeto superior u objeto origen (moverse hacia el tronco del árbol): insert+8 del teclado numérico
  • Moverse al objeto anterior (el objeto directamente antes de éste , en el mismo nivel): insert+4 del teclado numérico
  • Moverse al siguiente objeto ( el objeto inmediatamente después de éste, en el mismo nivel): insert+6 del teclado numérico
  • Moverse al primer objeto inferior ( la primera rama que sale de éste): insert+2 del teclado numérico
  • Moverse al foco: insert+tecla menos del teclado numérico
  • Activar objeto actual (Ej. Presionar enter, clic o Doble clic sobre el mismo): insert+enter del teclado numérico
  • Mover el ratón al objeto actual: insert+tecla dividir del teclado numérico
  • Moverse al ratón: insert+tecla multiplicar del teclado numérico

Cuando te mueves con el foco, el objeto actual del navegador cambia junto con él. Sin embargo, si deseas colocar el navegador de objetos en un objeto en especial, no cambia el foco, puedes desactivar FocoMueveNavegadorDeObjetos con insert+7.

7.3. Revisando el texto en el objeto actual

Algunas veces puedes querer saber como se deletrea algo, o probablemente puedas necesitar leer algo por palabras. Puedes revisar el texto del objeto activo con las siguientes órdenes:

  • Línea superior: insert+7 del teclado numérico
  • Línea anterior: tecla 7 del teclado numérico
  • Línea actual: tecla 8 del teclado numérico
  • Línea siguiente: tecla 9 del teclado numérico
  • Línea inferior: shift+9 del teclado numérico
  • Palabra anterior: tecla 4 del teclado numérico
  • Palabra actual: tecla 5 del teclado numérico
  • Palabra siguiente: tecla 6 del teclado numérico
  • Inicio de línea: shift+1 del teclado numérico
  • Carácter anterior: tecla 1 del teclado numérico
  • Carácter actual: tecla 2 del teclado numérico
  • Carácter siguiente: tecla 3 del teclado numérico
  • Fin de línea: shift+3 del teclado numérico
  • Leer todo el texto: shift + tecla más del teclado numérico

para la mayoría de los objetos , el texto contenido está conformado por nombre , valor y descripción. Sin embargo, para los objetos que actualmente contienen texto real o contenido editable, el texto actual es el texto en el objeto.

Por ejemplo: un objeto de una lista, tendrá texto que contenga su nombre, valor y descripción, pero en un campo de edición o en la consola del DOS de Windows, tendrá el texto actual que esté en el campo de edición o en la ventana.

Cuando un objeto contiene un cursor de edición (cursor de sistema), y se mueve, el cursor de revisión es también actualizado a esa posición. Si quieres que el cursor de revisión se quede donde está, y no se mueva con el cursor del sistema, you puedes desactivar CursorDeSistemaMueveCursorDeRevisión presionando insert+6.

7.4. Navegando con el cursor del sistema

Cuando el foco está sobre un objeto que está en forma de cursor de edición (sistema resaltado), puedes desplazarte por él normalmente como gustes. el NVDA te anunciará que te estás moviendo por caracteres, palabras, líneas y también anunciará si el texto está seleccionado o no.

Cuando te mueves con el sistema resaltado, la posición del cursor de revisión también se actualizará para coincidir con el sistema resaltado.

También puedes leer todo el texto (verbalizar todo) usando las teclas insert+flecha abajo. Nota sin embargo, que esto es diferente a utilizar insert+ tecla más que solo mueve el cursor de revisión; esto mueve el sistema resaltado como va.

7.5. Navegando con el Ratón

Cuando mueves el ratón, el NVDA puede verbalizar el objeto que se encuentre justo debajo del puntero del ratón si lo configuras para ello.

Incluso usando el ratón una persona ciega puede hacerse una idea de cómo está dispuesta la pantalla. el NVDA solo puede leer el objeto que está debajo del ratón, aún no puede leer información textual específica o detalles característicos.

8. Modos Virtuales

Los documentos complejos, como por ejemplo las páginas Web, están representadas en el NVDA como un modo virtual.

Un modo virtual, es una representación en texto plano de un documento que le permite al usuario desplazarse por el mismo, leer el texto y los campos que estén en el documento. Aparte de esto , también puedes presionar enter o barra espaciadora sobre enlaces o botones para activarlos, presionar enter o barra espaciadora en campos de edición para modificar su contenido, presionar enter o barra espaciadora en cuadros de verificación o botones de opción para seleccionarlos y presionar enter o barra espaciadora en cuadros combinados para habilitar el cambio de selecciónn de opciones.

Cuando te desplazas por un campo (como por ejemplo un enlace) se verbaliza el tipo de campo; sin embargo esta información no está físicamente representada en el modo virtual, lo que significa que puedes desplazarte normalmente a través del texto.

Actualmente el NVDA posee modos virtuales para mozilla FIRE fox y para Internet Explorer. El modo virtual tiene la capacidad de actualizarse dinámicamente, pero actualmente solo mozcilla FIRE fox hace uso de dicha característica. Aunque probablemente Internet Explorer muestre correctamente mas páginas, mozilla FIRE fox es mucho mejor con el contenido que cambia debido a java script etc. etc.

A causa de la complejidad de los modos virtuales, el NVDA puede tomarse un poco de tiempo para cargar algunas páginas. algunas se cargarán rápidamente, pero otras algo mas grandes tomarán algo mas de tiempo, cerca de 5 segundos o más.

8.1. Navegación con una simple letra

Para una navegación mas rápida, el NVDA también provee de unas teclas individuales para saltar a ciertos campos en el modo virtual. Por si mismas , saltan al campo siguiente y con la tecla shift saltan al campo anterior. Teclas:

  • encabezado: h
  • lista: l
  • elemento de lista: i
  • tabla: t
  • enlace: k
  • párrafo: p
  • campo de formulario: f

8.2. Formularios y modo pasar através del modo virtual

Algunas veces puedes desear interactuar con algunos formularios sin la combinación del modo virtual; de alguna manera, el modo virtual atraviesa exactamente por este problema y puede alternarse de activado a desactivado presionando insert+ barra espaciadora.

Sin embargo, generalmente cuando se presiona sobre un formulario este se activa automáticamente y cuando se carga una nueva página, se desactiva.

9. Configurando NVDA

9.1. Opciones de la Interface del Usuario

El cuadro de diálogo de las opciones del NVDA se encuentra en el menú preferencias. Contiene las siguientes opciones:

Idioma

Un cuadro combinado que te permite seleccionar el idioma que debería ser mostrado por la interface del Usuario y los mensajes de NVDA. Hay muchos idiomas, no obstante la última elección en la lista es llamada "Windows". esta elección dice a NVDA que utilice el idioma en el que Windows esté actualmente (en las opciones regionales y de idioma en el panel de Control). Por favor nota que NVDA debe ser reiniciado cuando se cambie el idioma, NVDA te preguntará si deseas reiniciar si haces un cambio de selección y presiona OK.

Ocultar la ventana de interface al Arrancar

Una casilla de verificación que dice a NVDA que no mantenga visible la interface de usuario, cuando arranque. Si la verificas será ocultada, si la desverificas, permanecerá visible hasta que el usuario la oculte manualmente con alt+f4.

Guardar la configuración al salir

Una casilla de verificación que cuando se verifica dice a NVDA que guarde automáticamente la configuración actual cuando salgas de NVDA.

9.2. Selección de Sintetizador

El cuadro de diálogo de sintetizador, que se encuentra en "Sintetizadores..." en el Menhú Preferencias, te permite seleccionar qué sintetizador debería utilizar NVDA para hablar con él.

El cuadro de diálogo contiene un sencillo cuadro combinado, que lista todos los sintetizadores disponibles. Selecciona el sintetizador que quieras utilizar con las teclas de cursor, y luego presiona ok. Si hay un error al cargar el sintetizador, un cuadro de mensaje te alertará del hecho, y conservará el sintetizador anterior cargado.

La lista actual de sintetizadores soportados por NVDA es: eSpeak (incluido con NVDA), SAPI4, SAPI5, y Silencio.

El sintetizador eSpeak está integrado directamente en NVDA, y no depende de cualquier otro controlador especial o runtime SAPI para ser instalado. NVDA arranca utilizando eSpeak por omisión. Este sintetizador debería funcionar en cualquier sistema en el que funcione NVDA, así funcionará definitivamente cuando se ejecute NVDA desde una memoria USB o un CD, en cualquier sistema.

Actualmente NVDA tiene dos controladores para SAPI4. La gente envía diferentes resultados con ambos controladores, por favor pruébalos y utiliza el que funcione mejor para ti.

El controlador de sintetizador silencio no es que sea útil, a menos que quieras no tener ninguna voz mientras se ejecute NVDA.

Por favor nota que no importa cuántas voces diferentes SAPI4 o SAPI5 (o motores) tengas instalados en tu sistema, sólo los actuales sintetizadores SAPI4 y SAPI5 se presentarán en el cuadro de diálogo Sintetizadores. Actualmente para seleccionar uno de los motores (voces), selecciona o SAPI4 o SAPI5, y luego en el diálogo de Opciones de voz, puedes elegir la voz que quieras.

9.3. Opciones de Voz

El cuadro de diálogo Opciones de Voz, se encuentra en el menú Preferencias, tiene las siguientes opciones:

Voz

Un cuadro combinado, listando todas las voces del sintetizador actual. Puedes utilizar las teclas de cursor para escuchar todas las varias elecciones.

Velocidad

Una barra de desplazamiento que va desde 0 a 100, (siendo 0 la velocidad más lenta y siendo 100 la más rápida). También puedes cambiar la velocidad de la voz desde cualquier sitio, utilizando las teclas insert+RePag e insert+AvPag para subir o bajar la velocidad respectivamente.

Tono

Una barra de desplazamiento que va desde 0 a 100, (siendo 0 el tono más bajo y siendo 100 el más alto).

Volumen

Una barra de desplazamiento que va desde 0 a 100, (siendo 0 el volumen más bajo y siendo 100 el más alto).

Verbalizar toda la puntuación

Una casilla de verificación, que cuando está verificada dice a NVDA que verbalice todos los símbolos de puntuación como palabras (muy útil para lectura exacta). Cuando está desverificada, NVDA deja la puntuación sin alterar, así el sintetizador aún leerá frases con la inflexión correcta, sin embargo los símbolos no deberían ser verbalizados en voz alta.

Decir cap para mayúsculas

Una casilla de verificación, que cuando está verificada dice a NVDA que verbalice la palabra "cap" antes de cualquier letra en mayúsculas, cuando pasamos con las flechas sobre ella o la verbalizamos cuando está siendo escrita. Normalmente, NVDA aumenta el tono ligeramente para cualquier letra en mayúsculas, pero algunos sintetizadores no podrán soportar esto bien, quizás esta opción podrá ser de uso.

9.4. Eco del Teclado

Este cuadro de diálogo se encuentra en el menú Preferencias, en "Eco del Teclado...". Contiene las siguientes opciones:

Verbalizar carácteres al escribir

Una Casilla de verificación que cuando está verificada significa que NVDA anunciará todos los carácteres que escribas en el teclado. Puedes también configurar esta opción desde cualquier sitio presionando insert+2.

Verbalizar palabras al escribir

Una casilla de verificación que cuando está verificada significa que NVDA anunciará todas las palabras al escribir en el teclado. Puedes también configurar esta opción desde cualquier sitio presionando insert+3.

Verbalizar teclas de órdenes

Una casilla de verificación que cuando está verificada significa que NVDA anunciará todas las teclas que no son carácteres al escribir en el teclado. Esto incluye combinaciones de teclas tales como control más cualquier otra letra. Puedes también configurar esta opción desde cualquier sitio presionando insert+2.

9.5. Opciones de ratón

El cuadro de diálogo de las opciones del ratón se encuentra en el menú Preferencias en "Opciones de Ratón...". Contiene las siguientes opciones:

Anunciar cambios en el formato del ratón

Una casilla de verificación, que cuando está verificada significa que NVDA anunciará los cambios de la forma del puntero del ratón cada vez que cambie. El puntero del ratón en Windows cambia de forma para comunicar cierta información tal como cuando algo es editable, o cuando algo se está cargando, etc.

Anunciar objetos bajo el ratón

Una casilla de verificación que cuando está verificada significa que NVDA anunciará el objeto (control) que actualmente está bajo el puntero del ratón, cuando lo mueves por la pantalla. Esto te permite encontrar cosas en la pantalla, moviéndo físicamente el ratón , antes que intentar encontrarlas a través de la navegación de objetos. podría no funcionar correctamente en todas las ocasiones, pero podría llegar a ser utilizable por algunas personas. También puede ser activado o desactivado desde cualquier sitio con insert+m.

9.6. Opciones de presentación Objeto

Se encuentra en el menhú Preferencias en "Opciones de Objeto..." Este cuadro de diálogo contiene las siguientes opciones:

Anunciar Herramientas de Ayuda

Una casilla de verificación que cuando está verificada dice al NVDA que anuncie las herramientas de ayuda cuando aparezcan. Muchas ventanas y controles muestran un pequeño mensaje (o herramienta de ayuda) cuando mueves el puntero del ratón sobre ellos, o algunas veces cuando mueves el foco a ellos.

Anunciar Globos de Ayuda

Esta casilla de verificación cuando está verificada dice al NVDA que anuncie los globos de ayuda cuando aparezcan. Los globos de ayuda son como las herramientas de ayuda, pero normalmente son más grandes, y están asociados con eventos del sistema tales como un cable de red es desenchufado, o quizás para alertarte acerca de cuestiones de seguridad de Windows.

Anunciar teclas rápidas de los objetos

Cuando esta casilla de verificación está verificada, NVDA incluirá la tecla rápida que está asociada con un determinado objeto o control cuando es anunciado. Por ejemplo el menú Archivo en una barra de menú podría tener una tecla rápida alt+a.

Anunciar Nombres de Grupo de objeto

Algunos objetos o controles en los cuadros de diálogo están colocados en agrupaciones. Cuando esta casilla de verificación está verificada, NVDA anunciará en primer lugar el nombre del grupo, antes de anunciar el objeto. Algunas veces escuchando el nombre del grupo te ayuda a identificar para qué es el objeto, sin embargo algunas veces los grupos pueden llegar a ser molestos, así que puedes desactivarlo aquí.

Decir el Estado del Objeto Primero

Cuando esta casilla de verificación está verificada, NVDA anunciará el estado de un objeto antes cualquier otra información. Por ejemplo un botón con un nombre "reproducir" y un estado "presionado" será anunciado como "presionado reproducir botón" antes que "reproducir botón presionado". Esto es útil algunas veces si estás tabulando por muchos objetos del mismo tipo, pero sólo estás interesado realmente en ver si están "presionados", o "cverificados" etc.

Pitar en Barras de progreso

Una barra de progreso es un control que se asemeja un poco a una regla. Se usa para denotar cuando una tarea ha sido completada. También muestran un porcentaje, Para decirte numéricamente cómo de lejos está. Las barras de progreso muestran cosas tales como la carga de una página web, mirar el correo, o el procesamiento de un fichero de sonido, etc. Si esta casilla de verificación está verificada, NVDA utilizará pitidos para permitirte saber cuán lejos está una barra de progreso cuando se está moviendo. Esto es algunas veces mucho más agradable que escuchar montones de números.

9.7. Opciones del Modo Virtual

El cuadro de diálogo del modo virtual puede ser encontrado en el menú Preferencias, en "Modo Virtual...". Contiene las siguientes opciones:

Máximo número de Carácteres por línea

Este campo fija la longitud máxima de una línea de un modo virtual (en carácteres). No está garantizado que todas las líneas terminarán con esta longitud o por debajo, pero la mayoría debería.

Número máximo de líneas por página

Si bien los modos virtuales realmente no tienen páginas, este campo fija la cantidad de líneas que moverás cuando presiones avance o retroceso de página mientras se esté en un modo virtual.

Anunciar la presentación virtual cuando el foco cambia

Esta casilla de verificación cuando está verificada dice al NVDA que anuncie el texto y la presentación de información en el modo virtual para un control, cuando consigue el foco, o más que sólo pregunta a Windows por la información del control. Por ejemplo, tabulando a un enlace llamado "gatos" dirá "enlace gatos" más que "gatos enlace" porque en los modos virtuales cuando se navegan con las flechas, el tipo siempre es dicho antes que el texto. Podría ser deseable en ciertos casos no tener esta opción activada puesto que esto significa que el control será anunciado en algún orden mejor. Sin embargo, para algunos navegadores de internet, podrías obtener un anunciado de la información más exacto cuando esto esté activado.

actualizar el contenido dinámicamente

Cuando esta casilla de verificación está verificada, los modos Virtuales tienen la capacidad para ser actualizados según la página cambia, debido a cosas como java script. Actualmente NVDA solamente soporta actualización dinámica para Mozilla Firefox, pero tenemos la esperanza de que Internet Explorer también será soportado con esta característica en futuras revisiones.

Opciones de aviso de campos

Las otras once casillas de verificación en este cuadro de diálogo te permiten configurar qué tipo de información es anunciada cuando navegues con las flechas por los modos virtuales. Por ejemplo, anunciar enlaces, dirá la palabra "enlace" antes de cualquier texto que sea un enlace cuando esté verificada.

Puedes configurar enlaces, encabezados, gráficos, listas, elementos de lista, tablas, formularios, campos de formulario, citas, marcos y párrafos.

9.8. Opciones de formato de documento

Este cuadro de diálogo se encuentra en el menú Preferencias, en "Formateado de Documento...". Todas las casillas de verificación en este cuadro de diálogo son para configurar qué tipo de formateo deseas escuchar automáticamente cuando muevas el cursor a lo largo de los documentos como en Wordpad o Microsoft Word. Por ejemplo, si verificas la casilla de verificación anunciar nombre de fuente, cada vez que navegando en el texto sobre una fuente diferente, el nombre de la fuente será anunciado.

Puedes configurar nombre de fuente, tamaño de fuente, atributos de fuente, alineamiento de texto, estilo de texto, tablas, números de página, y números de línea.
