1 introducción

 NVDA , es un lector de pantalla para el sistema operativo Windows, que le permite al usuario acceder al computador sin la necesidad de visión alguna. Consultando al sistema operativo dónde se encuentra el usuario, qué ventanas y comandos están en la pantalla  y observando de cerca los cambios del sistema operativo, el  NVDA  puede procesar y hablar a través del sintetizador de voz esta información, de manera que el usuario entienda qué pasa exactamente
en la pantalla.

 Actualmente el NVDA soporta los sintetizadores de voz compatibles con sapi 5 y usa una estructura modular que permite soportar otros sintetizadores de voz que sean programados. Sapi 4 no está soportado pero se está trabajando actualmente para que lo sea en un futuro próximo.

 NVDA se basa en el uso de Microsoft active  accesibility (msaa), llamadas a las API de Windows de bajo nivel  y modelos de objetos de aplicaciones específicas.

 NVDA no usa controladores de vídeo o conectadores con la pantalla..la ventaja de todo esto es que  el  NVDA puede trabajar con diferentes configuraciones de sistema y no depende de una configuración particular de color o de modo de pantalla de vídeo.  NVDA tampoco tiene que instalar ningún driver  especial o archivo que requiera reiniciar el PC antes de ejecutar el programa por primera vez .

 NVDA no soporta aún dispositivos braille, aunque estamos trabajando actualmente  en esto.

 Hay actualmente unas pocas traducciones de la interface del usuario de NVDA  a diferentes idiomas  pero van a aparecer más en un futuro.

 NVDA debe estar habilitado para leer un texto en cualquier  idioma, siempre y cuando usted tenga una voz sapi 5 que hable en dicho idioma.

 NVDA es un programa experimental  y está bajo desarrollo; también es de código abierto; esto significa que cualquiera puede ver el código fuente o cambiarlo a su gusto para satisfacer sus necesidades . Debido a esto puede haber errores y las cosas pueden no funcionar como se espera en todos los sistemas. 

 Pero según más y más personas prueban el NVDA y proveen de retroalimentación al usuario, el  NVDA  debe ser capaz de crecer e incluso mejorar, actuando como una solución completamente gratuita para usuarios de Windows que sean ciegos o deficientes visuales.

 NVDA trabaja con muchos programas y solo depende de cuan accesibles sean éstos, para el buen funcionamiento del  NVDA   con ellos.

 A parte de las funcionalidades generales de Windows, el  NVDA funciona con Word pad, block de notas,outlook expres e internet explorer. También tiene soporte básico    para Microsoft Word 2000/xp/2003 y   Microsoft excel 2000/xp/2003. Aún no ha sido probado con Microsoft Office 2007.

 NVDA también tiene extensiones para Mozilla fire fox 2.0 y superiores , siendo el explorador recomendado para obtener mayor accesibilidad con el NVDA.

 Esta guía del usuario no es de ningún modo  una guía completa  para usar el  NVDA   y Windows  pero  si la información necesaria para que usted comience a usar el  NVDA  , navegar  y aprender por usted mismo como hacer muchas otras cosas.

 2 requisitos delsistema.

 NVDA corre en Windows xp y Windows vista, también corre en Windows 2000 pero no muchas personas han probado esto.

 NVDA no funcionará en Windows 95/98/me, ni trabaja aún en versiones de Windows de 64 bits.

 Usted debe tener como mínimo 256 de memoria ram; el NVDA correrá mejor en PC que posean 1.0 ghz de velocidad o superiores. Aunque el NVDA correrá en máquinas mas lentas, su velocidad de respuesta y actividades mas intensas como la carga de páginas web por ejemplo, pueden darse muy lentamente  

 3 obteniendo el NVDA .

 Si usted aún no tiene una copia del programa , puede descargarla visitando la web en la siguiente dirección :

 Http://www.nvaccess.org/NVDA/ 

 Vaya al encabezado de descargas, allí encontrará un link para descargar  la última revisión  del programa. 

 Las descargas del  NVDA están actualmente numeradas por revisión.  Esta revisión coincide exactamente con las veces que ha sido cambiado el código fuente. Sin embargo , solo porque se coloque una descarga en el Sitio web, no significa que esté completamente libre de errores, solo significa que los desarrolladores
sienten que las características actuales del NVDA parecen estar funcionando bastante bien en sus propios sistemas  y que probablemente sea hora de que el NVDA sea probado por el resto del mundo.

 El archivo con extensión ZIP que usted descargue, necesita ser descomprimido en cualquier carpeta de su sistema.

 NVDA normalmente se ejecutará en cualquier sector de la unidad de disco duro , pero también puede hacerlo desde una memoria USB o incluso desde un CD con suficiente espacio.

 4 iniciando  NVDA  .

 para inicializar el NVDA usted debe presionar enter  o hacer doble clik sobre el archivo llamado NVDA.exe ubicado en el directorio NVDA donde descomprimió el archivo con extensión ZIP.

 Cuando el NVDA arranque, usted escuchará primero  una serie de sonidos ascendentes que le indican que el programa se está cargando. Usted entonces debe escuchar que el NVDA verbalice “ NVDA activado". . dependiendo de cuan rápido sea su computador o de si se está ejecutando el programa desde una memoria USB o algún otro medio  mas lento, podría tomar  unos instantes para iniciar.  Si se está tomando un tiempo extra mas largo, el NVDA  debe verbalizar
“cargando subsistema, aguarda por favor”. 

 Si usted no escucha nada de esto, o escucha el sonido de error de Windows, o una serie de sonidos descendentes, significa que el NVDA presenta un error; es probable que usted necesite reportar el error a los desarrolladores. Por favor revise la página Web de NVDA para saber como hacer esto.

 Una vez que el NVDA arranque exitosamente, debe traer al frente su  ventana de interface , que le permitirá controlar las configuraciones del programa, y salir del mismo. 

 5 la interface del usuario .

 El NVDA posee una ventana de interface que contiene una barra de menús, con menús que le permitirán hacer cambios en la configuración del programa y salir del mismo. También tiene una guía rápida con una lista con la mayoría de los comandos de teclado que puede usar mientras esté ejecutando el NVDA.

 La ventana de la interface del NVDA es un buen lugar para acostumbrarse a navegar con el programa, dado que tiene un documento por donde se puede desplazar con las flechas,   además de los muchos menús y cuadros de diálogo.

 5.1 mostrando y ocultando la interface del NVDA.

 Aunque la ventana de la interface del NVDA aparece cuando usted inicia el programa,  no tiene que estar todo el tiempo navegando por ella. No hay nada peor que tener demasiadas aplicaciones abiertas y tratar de encontrar una en particular a través del uso de alt+ tab. 

 Para esconder la ventana de interface del NVDA solo debe presionar la combinación alt+f4 estando dentro de la misma ventana; esto no lo hará salir del NVDA, solo  esconderá la ventana de la interface.

 Para mostrar la ventana en cualquier momento, sólo debe presionar la combinación insert+n, o dar enter o doble clic sobre el icono en la bandeja de sistema de Windows.

 5.2 guardando y recuperando la configuración del NVDA.

 El NVDA guarda sus configuraciones en un archivo llamado NVDA.ini, probablemente usted nunca tenga que tocar dicho archivo, pero para usuarios avanzados, este archivo le permite cambiar algunas configuraciones    extras que aún no tienen los menús o cuadros de texto para hacerlo.

 El NVDA no guarda las configuraciones automáticamente, así que si usted sale del programa y lo reinicia , las mismas volverán a los valores originales.  para guardar los cambios usted debe escoger la opción guardar del menú NVDA  presionando enter sobre la misma, o en su defecto presionar las teclas control+s  estando en la ventana de interface del NVDA.

 Si el NVDA está en un sistema de solo lectura como un CD por ejemplo, no estará disponible la opción de guardar la configuración y este le dirá eso.

 Si usted cometió un error con la configuración y necesita regresar a la configuración guardada, puede hacerlo seleccionando la opción "regresar a la configuración guardada" ubicada en el menú  NVDA , o puede pulsar las teclas control+r estando en la ventana del NVDA.

 5.3  configuraciones de la interface del usuario.

 El cuadro de diálogo de las configuraciones del NVDA se encuentra en el menú preferencias , esto le permite a usted configurar las opciones acerca de si la ventana de interface del NVDA  se muestra al iniciar, o si usted quiere que se guarden las configuraciones al salir. También le permite cambiar el idioma de la interface y de los mensajes.

 5.4 sintetizadores.

 La opción de sintetizadores en el menú preferencias le permite a usted  seleccionar que sintetizador usar. Actualmente el NVDA solo soporta el sintetizador sapi 5, así que usted no necesitará usar esto, pero  en el futuro ahí es donde usted tendrá disponible la opción de seleccionar varios sintetizadores.

 5.5 configuración de voz.

 La configuración de voz en el menú preferencias, trae al frente un cuadro de diálogo que le permite cambiar la voz que usará el NVDA así como cambiar la velocidad de verbalización y el volumen de la voz.

 La velocidad de verbalización puede cambiarse desde cualquier lugar del sistema presionando las teclas insert+pág arriba o insert+pág abajo. insert +pág arriba aumentará en 5% e insert+pág abajo disminuirá en 5% la velocidad de verbalización.

 Allí hay dos opciones más que son: verbalizar signos de puntuación y decir  cap antes de las mayúsculas.

 La verbalización de la puntuación le permite escoger si el NVDA  verbalizará los signos de puntuación o si solo los usará  para controlar el modelo de verbalización.

 La opción de verbalizar cap antes de caracteres en mayúsculas, le dice al NVDA  que debe decir la palabra cap antes de verbalizar cualquier carácter en mayúsculas. Usualmente el NVDA levantará el tono de la voz para las letras mayúsculas pero no todas las voces pueden soportar esto, por lo que puede ser útil activar esta opción.

 5.6 eco de teclado.

 El NVDA puede verbalizar desde los caracteres y palabras que usted escriba , hasta las combinaciones de teclas que presione. todas éstas opciones son configurables desde   el eco de teclado (opción que se encuentra en el menú preferencias)

 Para cambiar estas opciones desde cualquier lugar del sistema, debe presionar  las siguientes combinaciones de teclas :

 Insert+2 del teclado alfanumérico para activar o desactivar verbalización de caracteres .

 Insert+ 3 del teclado alfanumérico para activar o desactivar verbalización de palabras e 
 Insert +4 del teclado alfanumérico para activar o desactivar verbalización de teclasde comando.

 5.7 configuración del ratón (mouse).

 Usted puede configurar el NVDA para anunciar cuando la posición del cursor cambie  (como por ejemplo cuando un programa se está cargando) o cuando usted mueve el ratón a cualquier campo de edición. También puede configurar el NVDA  si quiere  que automáticamente  verbalice lo que hay actualmente debajo del ratón, como su movimiento.

 Estas opciones se pueden encontrar en el cuadro de diálogo de configuración del Mouse (ratón) que está en el menú preferencias.

 5.8 presentación de los objetos.

 La opción de presentación de objetos, trae al frente un cuadro de diálogo que le permite configurar cómo el NVDA trata ciertos controles (u objetos) en Windows.

 Usted puede decirle al NVDA si desea leer las ayudas cuando éstas aparezcan , si  quiere el estado de un objeto (por ejemplo si está verificado, seleccionado, pulsado, etc. etc.) o si quiere que el NVDA  emita un pitido (señal sonora) cuando una barra de progreso se esté moviendo.

 6 navegando con el NVDA.

 6.1 navegando por focos.

 La manera más común de navegar por Windows con el NVDA es simplemente  moviéndose con los comandos normales del teclado,  como por ejemplo tab. o shift+tab. para adelantar o retroceder entre objetos., presionando Alt. para acceder a la barra de menús y luego navegar por ellos con las flechas, usando Alt. + tab. para moverse entre aplicaciones en proceso; haciendo esto, el NVDA anunciará  lo que  tiene el foco.
 
 Cuando el NVDA anuncia un objeto (Ej. cuando recibe el foco). Usualmente  dice el nombre del objeto , tipo, valor,descripción, atajo de teclado e información de su posición.

 Esto es bastante similar en otros lectores de pantalla, sin embargo los objetos pueden ser algo diferentes .

 6.2 navegación de objetos.

 Si usted desea navegar alrededor de algunos objetos pero sin mover el foco, usted puede usar la navegación por objetos del NVDA, esto le permitirá moverse objeto porobjeto, como en una estructura de árbol.

 La estructura en árbol de los objetos, comienza en el escritorio de Windows; el tronco del árbol. 

 las ramas principales del árbol. son las aplicaciones actualmente en proceso; y por supuesto éstas aplicaciones tienen sus propias ramas (los objetos) representando varias partes de la aplicación.

 La estructura puede volverse bastante larga pero eso le permite  navegar en un orden lógico ; así usted podrá encontrar rapidamente la información que necesita.

 Esta estructura no tiene nada que ver con la forma en que los objetos  se colocan en la pantalla , pero si con la forma como lógicamente se relacionan.  puede tomar un poco de tiempo acostumbrarse a esto, pero  una vez que usted  lo logre, usted debe estar preparado para moverse  fácilmente.  

 Actualmente el NVDA no puede navegar los objetos por el diseño de una pantalla debido a que no usa controlador de video  ni conectores de pantalla.

 El NVDA escoje no interactuar con la pantalla, mucho de esto porque sobre todo debe permitir  el despeje  y  una percepción más lógica de una aplicación y del sistema operativo, en lugar de tratar de obtener una imagen plana de la pantalla.

 No todas las aplicaciones tienen una estructura en árbol perfecta, pero  esto depende del desarrollador de la aplicación en particular; muchas parecen estar bien, ciertamente muchos cuadros de diálogo y aplicaciones altamente estructuradas parecen obtener derechos.

 Para navegar por objetos, use los siguientes comandos:

 Verbalizar objeto actual  insert+ 5 del teclado numérico.
 Movérse al  objeto superior u objeto origen ( moverse hacia el tronco del árbol) insert+8 del teclado  numérico.
 Moverse al objeto anterior (el objeto directamente antes de éste , en el mismo nivel) 
 insert+4 del teclado numérico.
 Moverse al siguiente objeto ( el objeto inmediatamente después de éste , en el mismo nivel) insert+6 del teclado numérico.
 Moverse al primer objeto  inferior ( la primera rama  que sale de éste) insert+2 del teclado numérico.
 Moverse al foco  insert+ tecla menos de la calculadora.
 Activar objeto actual (Ej. Presionar enter , clic  o Doble clic sobre el mismo)  insert+ enter de la calculadora.
 Mover el ratón (Mouse) al objeto actual insert+ tecla dividir de lacalculadora.
 Moverse al ratón (Mouse) insert+ tecla multiplicar de la calculadora.

 6.3 revisando texto en el objeto activo.

 Algunas veces usted puede querer saber como se deletrea algo, o probablemente pueda necesitar leer algo por palabras. Usted puede revisar el texto del objeto activo con los siguientes comandos :

 Línea superior insert+7 del teclado numérico.
 Línea anterior tecla 7 del teclado numérico
 Línea actual tecla 8 del teclado numérico 
 Línea siguiente tecla 9 del teclado numérico 
 Línea inferior shift+9 del teclado numérico
 Palabra anterior  tecla 4 del teclado numérico
 Palabra actual tecla 5 del teclado numérico
 Palabra siguiente tecla 6 del teclado numérico
 Inicio de línea shift+1 del teclado numérico
 Carácter anterior tecla 1 del teclado numérico
 Carácter actual tecla 2 del teclado numérico
 Carácter siguiente tecla 3 del teclado numérico 
 Fin de línea shift+3 del teclado numérico
 Leer todo el texto shift + tecla más del teclado numérico

 para la mayoría de los objetos , el texto  contenido está conformado por nombre , valor y descripción. Sin embargo, para los objetos que actualmente contienen texto real o contenido editable,  el texto actual es el texto en el objeto.

 Por ejemplo: un objeto de una lista, tendrá texto que contenga su nombre, valor y descripción, pero  en un campo de edición o en la consola  del DOS de Windows, tendrá el texto actual que esté en el campo de edición o en Windows.

 6.4 navegando con el sistema resaltado.

 Cuando el foco está sobre un objeto que está en forma de cursor de edición (sistema resaltado),  usted puede desplazarse  por él normalmente como guste.  el NVDA  le anunciará que usted se está moviendo por caracteres, palabras, líneas y también anunciará si el texto está seleccionado o no.

 Cuando usted se mueve con el sistema resaltado, la posición del cursor de revisión también se actualizará para coincidir con el sistema resaltado.

 Usted también puede leer todo el texto  (verbalizar todo) usando las teclas insert+ flecha abajo. Note  sin embargo, que esto es diferente a usar  insert+ tecla más que solo mueve el cursor de revisión; esto mueve el sistema resaltado como va.

 6.5 navegando con el ratón (mouse).

 Cuando usted mueve el ratón, el NVDA puede verbalizar el objeto que se encuentre justo debajo del puntero del ratón(Mouse) si lo configura para ello.

 Incluso usando el ratón (Mouse) una persona ciega puede obtener una idea de cómo está dispuesta la pantalla .. el NVDA  solo puede leer el objeto que está debajo del ratón, aún no puede leer información textual específica o detalles característicos.

 7 Modos virtuales.

 Los documentos complejos, como por ejemplo las páginas Web, están representadas en el NVDA  como un modo virtual.

 Un modo virtual, es una representación en texto plano de un documento que le permite al usuario desplazarse  por el mismo, leer el texto y los campos que estén en el documento. Aparte de esto , usted también puede presionar enter o barra espaciadora sobre enlaces o botones para activarlos, presionar enter o barra espaciadora en campos de edición  para modificar su contenido, presionar enter o barra espaciadora en cuadros de verificación o botones de
opción para seleccionarlos y presionar enter o barra espaciadora en cuadros combinados para habilitar   el cambio de selecciónn de opciones.

 Cuando usted se desplaza por un campo (como por ejemplo un enlace) se verbaliza el tipo de campo; sin embargo esta información no está físicamente representada en el modo virtual, lo que significa que usted puede desplazarse normalmente a través del texto..

 Actualmente el NVDA posee modos virtuales para mozilla FIRE fox y para Internet Explorer. El modo virtual tiene la capacidad de actualizarse dinámicamente, pero actualmente solo mozcilla FIRE fox hace uso de dicha característica. Aunque probablemente  Internet Explorer muestre correctamente mas páginas, mozilla FIRE fox es mucho mejor  con el contenido  que cambia debido a java script etc. etc.

 A causa de la complejidad de los modos virtuales, el NVDA  puede tomarse un poco de tiempo para cargar algunas páginas . algunas se cargarán rápidamente, pero otras algo mas grandes tomarán algo mas de tiempo , cerca de 5 segundos o mas.

 Para una navegación mas rápida, el NVDA también provee de unas teclas individuales  para saltar a ciertos campos en el modo virtual. Por si mismas , saltan al campo siguiente y  con la tecla shift saltan  al campo anterior..

 Estas teclas son las siguientes:

 Encabezados tecla h 
 listas tecla l 
 Objetos de listas tecla i 
 Tabla tecla t 
 Enlace tecla k 
 Párrafo tecla  p
 Campo de formulario tecla f 

 Algunas veces usted puede desear interactuar con algunos formularios sin la combinación  del modo virtual; de alguna manera, el modo virtual atraviesa exactamente por este problema  y puede alternarse de activado a desactivado presionando insert+ barra espaciadora.

 Sin embargo, generalmente cuando se presiona sobre un formulario este se activa automáticamente y cuando se carga una nueva página, se desactiva.

 8 pitidos (señales sonoras) de la barra de progreso.

 En Windows, la barra de progreso se usa para denotar cuando una tarea ha sido completada, como por ejemplo, copiar archivos en Internet Explorer,, cargar una página Web,  o revisar el correo en el Outlook Express, todo  esto usa una barra de progreso.

 El NVDA puede lanzar un pitido para indicarle donde se encuentra la barra de progreso. Si el pitido es alto y cercano, quiere decir que la barra está al 100%. Usted aprenderá pronto a distinguir según el pitido,  que tan cerca o que tan lejos está la barra de progreso.

 9 modo de pitido verbal.

 El modo de pitido verbal en el NVDA es un modo en el que en lugar de verbalizar todo, el NVDA  simplemente emite un pitido rápido y fuerte. Esto es muy útil cuando usted está realizando tareas donde hay muchas pantallas de actualización , como por ejemplo usando un programa para quemar CD desde la consola del DOS o copiando gran cantidad de archivos.

 En lugar de escuchar cada archivo o cada porcentaje, o en lugar de presionar constantemente la tecla control para callar la verbalización, usted puede presionar las teclas insert +s dos veces y activar  el modo de pitido verbal. Todo lo que usted escuchará, serán pequeños pitidos hasta que la pantalla se detenga lo que le indica que la tarea a finalizado.

 Para volver al modo normal , solo debe presionar insert+ s  nuevamente .
