1. Einführung

1.1 Allgemeine Features

NonVisual Desktop Access (kurz NVDA) ermöglicht blinden und sehbehinderten Menschen den Zugang und die Interaktion mit dem Betriebssystem Microsoft Windows, Microsoft Office und zahlreiche Anwendungen von Drittanbietern durch das Feedback der synthetischen Sprache sowie auch optional mit Braille-Unterstützung.

Die wichtigsten Highlights sind:

  • Vollständige Lauffähigkeit von einem USB-Stick oder einem tragbaren Medium ohne Installation
  • Einfache Bedienung der sprechenden Installation
  • Surfen im Web mit Mozilla Firefox 3
  • Verwalten von E-Mails mit Mozilla Thunderbird 2 (Version 3 befindet sich noch im Beta-Status!)
  • Frühe Unterstützung für Microsoft Internet Explorer 7
  • Wichtige Unterstützung für Microsoft Outlook Express 6 bzw. der Nachfolger, dem Windows Mail 7
  • Unterstützung für Microsoft Office-Anwendungen
  • Unterstützung für zugängliche Java-Anwendungen (Java Runtime Enviroment)
  • Frühe Unterstützung für Adobe Reader
  • Frühe Unterstützung für IBM Lotus Symphony
  • Unterstützung für Eingabeaufforderung und Konsolenanwendungen
  • Automatische Ankündigung von Text unter der Maus und optional hörbare Angaben der Mausposition
  • Braillezeilen-Unterstützung mit den Treibern von Freedom Scientific, HandyTech oder von BRLTTY

1.2 Internationalisierung

Es ist wichtig, dass die Menschen überall auf der Welt, egal welche Sprache sie sprechen, den gleichen Zugang zu Technologien erhalten. NVDA wurde bislang in über 20 Sprachen übersetzt, darunter: Brasilianisches Portugiesisch, Kroatisch, Tschechisch, Finnisch, Französisch, Galizisch, Deutsch, Ungarisch, Italienisch, Japanisch, Portugiesisch, Russisch, Schwedisch, Slowakisch, Spanisch, Traditionelles Chinesisch, Afrikanisch, Polnisch, Thai, Ukrainisch und Vietnamesisch

1.3 Unterstützung mit Sprachausgabe

Abgesehen von der Bereitstellung der Schnittstelle für Meldungen in mehreren Sprachen kann der Benutzer in NVDA Inhalte in jeder Sprache lesen, sofern dieser über eine Sprachausgabe über dessen Sprache verfügt.

NVDA ist standardmäßig mit einer freien multilingualen Sprachausgabe namens eSpeak ausgestattet. Darüber hinaus kann NVDA sowohl mit SAPI4- und SAPI5-Sprachen und auch mit der Audiologic Sprachausgabe gearbeitet werden.

1.4 Unterstützung mit Brailleausgabe

Für Nutzer, die eine eigene Braillezeile haben, kann NVDA die Informationen in Braille-Schrift ausgeben. Dabei werden von NVDA die Braillezeilen PAC Mate und Focus von Freedom Scientific sowie auch die von HandyTech unterstützt. Sofern vom BRLTTY-Paket ebenfalls unterstützt, werden auch noch viele andere Braillezeilen zur Verfügung stehen.

1.5 Neuerungen und Tests

Der NonVisual Desktop Access ist nicht zwingend an Marktinteressen und -trends gebunden. In kommerziellen Screen Reader werden bereits gute Funktionen eingebaut, aber manchmal nur, auf Grund von Markttrends. Anstatt für den Benutzer sinnvolle Funktionen zu implementieren. Möglicherweise läuft NVDA nicht so stabil, wie der eine oder andere kommerzielle Screen Reader, kann aber als Wergzeug zum Experimentieren und Testen verwendet werden. Ideen und Funktionen können einfach und schnell getestet werden, die bislang vielleicht nicht in anderen Windows Screen Readern Einzug gehalten haben. Ein Beispiel für die Fähigkeit von NVDA ist durch Signaltöne dem Benutzer mitzuteilen, dass sich ein Fortschrittsbalken füllt. Je höher der Signalton wird, desto näher rückt er der 100-Prozent-Marke. Diese Funktion wurde von einigen Benutzern ihrer Screen Reader durch selbstgeschriebene Skripte o. ä. hinzugefügt. Aber sie wurden jedoch nie offiziell in den kommerziellen Screen Readern mitausgeliefert.

1.6 Design und Anwendung

NVDA ist in der Programmiersprache Python geschrieben. Der Code ist durch seinen modularen Aufbau somit leicht erweiterbar, um die Zugänglichkeit der neuen Programme in Windows ergänzen zu können. Um allgemeine Unterstützung für eine bestimmte Anwendung zu erhalten, können sogenannte AppModule (Module für Anwendungen) geschrieben werden. Damit NVDA mit komplexen Dokumenten oder andere Daten umgehen kann, kann ein sogenannter Virtueller Ansicht geschrieben werden. Objekte für NVDA können geschrieben werden, um Unterstützung für spezifische Kontrollen in Windows zu erhalten. Dabei versucht NVDA immer die benötigten Elemente der Oberflächen für Programme und Betriebssystem so zugänglich wie möglich für den aktiven Benutzer zu erhalten. NVDA besitzt keine Filter-Funktion und gibt daher einfach die Informationen, die es finden kann, trotzdem einfach an den Benutzer weiter.

Für die Kommunikation mit dem Betriebssystem und Programmen verwendet NVDA eine Kombination aus Funktionen vom Betriebssystem selbst, der Microsoft Active Accessibility (kurz MSAA genannt), IAccessible2, Java Access Runtime Enviroment und programmspezifische Schnittstellen von bestimmten Anwendungen. Dabei verwendet NVDA keine speziellen Video-Intercept-Treiber, sondern wird immer versuchen, so viele Informationen aus Accessibility-spezifischen Schnittstellen wie möglich, vor dem Rückgriff auf andere Mittel, aufzugreifen.

3. Bezugsquellen

Falls Sie noch keine Version bereits haben, können Sie sie im Download-Bereich herunterladen.

Derzeit gibt es zwei Möglichkeiten: Die eine Variante ist die Archivdatei und die andere Variante das Installationsprogramm. Die ausführbare Archivdatei können Sie einfach auf einen USB-Stick in einem Ordner Ihrer Wahl entpacken und sofort starten. Während beim Installer wird NVDA auf Ihrem System installiert und anschließend können Sie es aus dem Startmenü aus aufrufen. Der Installationsvorgang läuft automatisch mit Sprache während des Installationsvorgangs.

4. Installation

4.1 Das Installationsprogramm

Wenn Sie bereits das Installationsprogramm haben, einfach Eingabetaste betätigen oder mit Doppelklick auf die Datei und der Installationsvorgang wird gestartet.

Während die Installation gestartet wird, hören Sie ein Sound-Logo. Danach wird eine temporär entpackte Kopie von NVDA gestartet. Diese erlaubt Ihnen den Anweisungen auf dem Bildschirm zu folgen. Sollten Sie eine frühere Kopie von NVDA benutzt haben, bevor Sie die Installation gestartet haben, erscheint eine Meldung, die besagt, dass diese nun beendet wird. Nach dem Betätigen der Eingabetaste beendet das Installationsprogramm die bereits laufende Version und startet eine eigene Kopie. Dies ist erforderlich, damit das Installationsprogramm alle Dateien erneuern kann.

Sobald Sie die Installation erfolgreich abgeschlossen haben, werden Sie vom Installer benachrichtigt, dass die Installation von NVDA erfolgreich fertig gestellt wurde. Sie werden gebeten, das System neu zu starten. Es ist sehr wichtig, dass Sie das System auch neu starten. Ansonsten könnte es zu unerwünschte Folgen führen.

4.2 Die Archivdatei

Wenn Sie die Archivdatei bereits haben, entpacken Sie alle Dateien in ein Ordner Ihrer Wahl auf der Festplatte oder auf einem tragbaren Medium.

5. Erste Schritte mit NVDA

5.1 NVDA starten

Wenn Sie NVDA bereits installiert haben, können Sie entweder es vom Desktop aus mit Steuerung+Alt+N starten oder rufen Sie einfach das Menü aus dem Startmenü unter Programme auf.

Öffnen Sie den Ordner, wo Sie die Archivdatei bereits entpackt haben und starten Sie einfach mit Eingabetaste oder mit Doppelklick auf die Datei "NVDA.exe".

Wenn NVDA gestartet wird, hören Sie erst eine aufsteigende Reihenfolge von Signaltönen (zur Signalisierung, dass NVDA im Hintergrund gestartet wird), gefolgt von einer Meldung, dass NVDA jetzt gestartet ist. Je nachdem, wie schnell Ihr Computer ist oder wenn Sie NVDA von einem USB-Stick oder einem anderen langsameren Medium starten, so kann dies eine Weile in Anspruch nehmen, bis der Ladevogang abgeschlossen ist. Wenn der Vorgang dennoch sehr lange dauert, erhalten Sie von NVDA die Meldung, dass benötigten Komponente eingelesen werden.

Falls Sie entweder gar nichts, von Windows einen Fehler-Sound, oder vom NVDA eine absteigende Reihenfolge von Signaltönen hören, bedeutet das, dass womöglich ein Fehler aufgetreten ist. Bitte schauen Sie auf der Homepage nach, ob ggf. etwas bekannt ist bzw. wie das Problem zu lösen ist. Ansonsten melden Sie es uns (über unser Kontaktformular auf der Startseite).

Wenn NVDA zum ersten Mal startet, wird Ihnen eine Dialogbox angezeigt, mit einigen grundlegenden Informationen über die entsprechende NVDA-Taste und dem NVDA-Menü (Bitte beachten Sie auch weitere Abschnitte dieser Themen). Das Dialogfeld enthält außerdem zwei Checkboxen (Kontrollfelder). Mit der ersten Checkbox können Sie estlegen, ob die Capslock-Taste (Dauergroßschreibtaste) als NVDA-Taste definiert werden soll. Mit der zweiten Checkbox legen Sie fest, ob der Willkommensdialog erneut angezeigt werden soll.

5.2 Über die NVDA-Tastenbefehle

Die meisten Befehle werden ausgelöst, indem die NVDA-Taste in Kombination mit einer oder mehreren Tasten gedrückt werden. Im Gegensatz dazu, werden die Text-Lesebefehle ausschließlich mit den Tasten des Nummernblocks ausgelöst. Die Tasten des Nummernblocks bekommen zur besseren Erkennung die Zeichenfolge "NUM" vorangestellt. Ein Beispiel: NUM 1 ist gleichbedeutend mit der Taste 1 des Nummernblocks.

Standardmäßig sind die Einfügetasten des Nummernblocks und der erweiterten Tastatur als NVDA-Tasten voreingestellt, wobei es auch so eingestellt werden kann, dass die Dauergroßschreibtaste als NVDA-Taste zur Verfügung steht. Wenn Sie die Dauergroßschreibtaste als NVDA-Taste benutzen, wird ein einmaliges drücken bzw. Das Halten der Taste von NVDA verwendet. Drücken Sie die Dauergroßschreibtaste zwei mal schnell, führt sie ihre eigentliche Funktion aus.

Viele Tastenbefehle von NVDA sind nicht in diesem Handbuch aufgelistet, sondern stehen im Handbuch der "Kurztasten- und Befehlsreferenz".

Sie können aber mit NVDA+1 die Tastaturhilfe ein- bzw. ausschalten. Bei aktiviertem Modus können Sie beliebige Tasten und -Kombinationen drücken und NVDA nennt Ihnen deren Funktionen und Bedeutungen. Wird diese Taste bzw. Tastenkombination nicht von NVDA verwendet, wird die Funktion direkt ausgeführt.

6. Das NVDA-Menü

Im NVDA-Menü können Sie Einstellungen von NVDA ändern, die Einstellungen speichern bzw. wiederherstellen, die Hilfe aufrufen, das Wörterbuch für die Aussprache bearbeiten, das Protokoll sich anzeigen lassen oder NVDA komplett beenden.

6.1 Auf das NVDA-Menü zugreifen

Mit der Tastenkombination NVDA+N können Sie sich, wo immer Sie sich befinden, das Kontextmenü des NVDA-Symbols anzeigen lassen. Sie können sich im Menü mit den Cursortasten bewegen. Mit der Eingabetaste öffnen Sie die Menüpunkte. Im Konfigurationsmenü können Sie die Einstellungen nach Ihren Wünschen anpassen.

Das "Extras"-Menü enthält nützliche Einträge, wie die Anzeige des Protokolls und die Python-Konsole für Entwickler. Das Hilfe-Menü erlaubt Ihnen das Benutzerhandbuch, die Befehls- und Kurztastenreferenz und vieles mehr anzuzeigen. Weiterhin enthält das Menü einträge um die Einstellungen von NVDA zu speichern bzw. zurückzusetzen und um NVDA komplett zu beenden.

Das NVDA-Menü kann auch über einen Rechtsklick auf das NVDA-Symbol im Infobereich angezeigt werden. Alternativ können Sie auf der Tastatur Windows+B betätigen und mit den Cursor-Tasten zum NVDA-Menü navigieren.

6.2 Konfigurationen

6.2.1 Das Menü "Einstellungen"

6.2.1.1 Allgemeine Einstellungen

Das Dialogfeld für die Allgemeinen Einstellungen befindet sich im Menü "Einstellungen" (auch mit NVDA+Steuerung+G erreichbar). Betätigen Sie den Button (Schalter) "OK", um die Änderungen zu speichern oder den Button "Abbrechen" (oder auch die Escape-Taste), um die Änderungen zu verwerfen. Dieses Menü enthält die folgenden Optionen:

Sprache

Ein Auswahllistenfeld, in der Sie Ihre Sprache auswählen können, in der die Meldungen und die Benutzerschnittstelle in der von Ihnen ausgewählten Sprache angezeigt werden soll. Standardmäßig wird die Option "Standardbenutzer, Windows" ausgewählt. Dabei greift NVDA auf die "Regions- und Sprach-Einstellungen" in der Systemsteuerung zurück. NVDA bietet zahlreiche integrierte Sprachen an. Bitte beachten Sie, dass nach erfolgter Auswahl NVDA neu gestartet werden muss. Sie werden dazu aufgefordert, in dem Sie auf den "OK"-Button, um NVDA anschließend automatisch neu zu starten.

Konfiguration beim Beenden speichern

Diese Option ist eine Checkbox, bei der Sie NVDA anweisen, die aktuellen Einstellungen beim Beenden automatisch zu speichern.

Warnen, bevor NVDA beendet wird

Diese Option ist eine Checkbox, die es Ihnen ermöglicht zu entscheiden, ob beim Beenden von NVDA eine Meldung mit der Frage angezeigt werden soll, ob NVDA wirklich beendet werden soll oder nicht.

Protokolltyp

Dies ist eine Auswahlliste, in der Sie entscheiden können, wie umfangreich die Protokollierung im laufendem Betrieb von NVDA sein soll. Im Allgemeinen sollten Sie diese Option nicht benötigen, falls das Protokoll nicht ausreichend sein sollte. Allerdings, wenn Sie die zusätzlichen Informationen in einem Fehlerbericht angeben möchten, dann kann es eine sinnvolle Option sein für die Entwickler.

6.2.1.2 Auswahl der Sprachausgaben

Das Dialogfeld "Sprachausgabe", den Sie im Menü "Einstellungen" finden, ermöglicht Ihnen, eine Sprachausgabe auszuwählen, die NVDA zum Sprechen standardmäßig verwenden soll.

Dieses Dialogfeld enthält ein einfaches Auswahllistenfeld, in der alle verfügbaren Sprachausgaben angezeigt werden. Sie können hier eine Sprachausgabe Ihrer Wahl auswählen und mit der Eingabetaste diese aktivieren. Sollten beim Starten der Sprachausgabe Fehler auftreten, werden Sie mit einer Meldung darauf aufmerksam gemacht.

Momentan unterstützt NVDA folgende Sprachausgaben: eSpeak, SAPI4, SAPI5, Audiologic, Display und Silence.

Die Sprachausgabe eSpeak ist bereits in NVDA integriert und benötigt keine Treiber oder SAPI-Schnittstelle, um zu funktionieren. NVDA startet standardmäßig mit dieser Sprachausgabe. Es sollte auf jedem Computer lauffähig sein, auf dem NVDA funktioniert, dies gilt auch, wenn NVDA von einem USB-Stick oder einem anderen Medium ausgeführt wird.

Die SAPI4-Option ermöglicht die Nutzung der SAPI-Sprachausgabe. Sofern zu SAPI4 auch die passenden Stimmen auf Ihrem Computer installiert sind.

Die SAPI5-Sprachausgaben verwenden die SAPI-Stimmen von Microsoft. Bei Windows XP ist es Microsoft Sam, bei Windows Vista ist es Microsoft Anna und bei anderen ist es Microsoft May. Je nachdem, welche Stimmen Sie installiert haben.

Die Sprachausgabe "Display" ist für sehende Entwickler sinnvoll, die wissen möchten, was NVDA gerade spricht, wenn Sie es mit Ihren Anwendungen testen.

Die Sprachausgabe "Silence" ist für Benutzer mit einer Braillezeile möglicherweise nützlich, da die Sprache komplett ausgeschaltet wird. Bitte beachten Sie, dass nur die aktuell zu Verfügung stehenden Sprachausgaben angezeigt werden. Egal, wie viele SAPI4- oder auch SAPI5-Treiber installiert sind. Die zugehörigen Stimmen, können Sie im Dialogfeld "Klangeinstellungen" auswählen.

Zu berücksichtigen ist, dass nur die aktuell zur Verfügung stehenden Sprachausgaben angezeigt werden. Dabei ist es nicht relevant, wie viele Treiber für SAPI4- bzw. SAPI5-Stimmen installiert sind. Die zugehörigen Stimmen können Sie in den Klangeinstellungen festlegen.

6.2.1.3 Klangeinstellungen

Zu finden sind die Einstellungen im Menü "Optionen" oder auch mit NVDA+Steuerung+V zu erreichen. Hier können Sie Änderungen an der aktuell ausgewählten Sprachausgabe vornehmen. Viele der enthaltenen Einstellungen, können jederzeit über die Klangeinstellungen geändert werden. Mit NVDA+Steuerung+Pfeil nach links oder Pfeil nach rechts können Sie eine Einstellung entsprechend auswählen, und den Wert nach Ihren Wünschen mit NVDA+Steuerung+Pfeil nach oben oder Pfeil nach unten anpassen.

Die folgenden Einstellungsmöglichkeiten gibt es:

Stimme

Dieses Auswahllistenfeld stellt die Stimme der aktuell ausgewählten Sprachausgabe ein. Mit Pfeil nach oben und Pfeil nach unten können Sie sich durch die Liste der Stimmen bewegen und die Stimmen anhören, bevor Sie ihre Auswahl festlegen.

Variante

Diese Ausklappliste ermöglicht die Variante zur Sprachauswahl der zu verwendenden Sprachausgabe. Zur Zeit wird diese Option lediglich von eSpeak unterstützt. Die Varianten von eSpeak werden nicht als Stimmen bezeichnet, sondern besitzen lediglich andere Eigenschaften.

Geschwindigkeit

Diese Option legt fest, wie schnell oder langsam die Sprechgeschwindigkeit sein soll. Dabei kann von 0 (langsamste) bis 100 (schnellste) eingestellt werden.

Stimmenhöhe

Diese Option legt fest, wie hoch oder tief die STimmhöhe sein soll. Dabei kann von 0 (tiefste) bis 100 (höchste) eingestellt werden.

Lautstärke

Diese Option legt fest, wie die Lautstärke sein soll. Dabei sind Werte von 0 bist 100 Prozent möglich. Wobei 100 die maximale Lautstärke entspricht.

Betonung

Diese Option legt fest, wie stark oder schwach die Betonung sein soll. Momentan kann diese Einstellung nur in eSpeak vorgenommen werden.

Alle Satzzeichen ansagen

Diese Checkbox legt fest, ob von NVDA alle Satzzeichen angesagt werden sollen oder ob nur zur Steuerung der Satzmelodie es verwendet werden soll.

Stimmveränderung bei Großbuchstaben

Die Checkbox legt bei fest, dass die Stimme bei Großbuchstaben angehoben werden soll.

Vor Großbuchstaben "groß" ansagen

Diese Checkbox legt fest, ob beim Navigieren oder beim Schreiben von Großbuchstaben das Wort "groß" vor Großbuchstaben angesagt werden soll. Normalerweise hebt NVDA bei Großbuchstaben die Stimme an. Es gibt jedoch Sprachausgaben, die diese Funktion nicht unterstützen. Für solche Sprachausgaben kann es sinnvoll sein, diese Option einzuschalten.

Signaltöne vor Großbuchstaben

Diese Checkbox stellt eine Alternative zur vorherigen Option dar, nur mit dem Unterschied, dass bei Großbuchstaben ein kurzer Signalton ausgegeben wird.

6.2.1.4 Braille-Einstellungen

Dieses Dialogfeld kann über das Konfigurationsmenü und dem Menüeintrag "Braille-Einstellungen" erreicht werden.

Braillezeile

Die erste Option, wenn Sie in dieses Dialogfeld kommen, ist das erste Feld die Auflistung der Braillezeilen. Es stehen drei Optionen zur Auswahl. Jede dieser ist mit den Cursor-Tasten zu erreichen.

"Keine Braillezeile" meint, dass Sie keine Braillezeile haben oder diese nicht angeschlossen ist.

"Freedom Scientific" ist für die Brailezeilen Fokus oder PAC Mate von Freedom Scientific. Beachten Sie, dass diese Option nur angezeigt wird, wenn Sie die Treiber für einer der Braillezeilen von Freedom Scientific installiert haben. Um weitere Informationen über die Bezugsquelle und die Installation der Treiber von Freedom Scientific zu erfahren, gehen Sie auf die Seite http://www.freedomscientific.com/PACMATE-HQ/FS_Braille_Display_Driver.asp.

"HandyTech" ist für die Braillezeilen von HandyTech. Beachten Sie, dass diese Option nur dann funktioniert, wenn Sie die Universal-Treiber von HandyTech installiert haben. Die Treiber können unter folgender Adresse bezogen werden: ftp://ftp.handytech.de/public/Software/BrailleDriver/bsd1111a.exe.

"BRLTTY" kann mit einer Vielzahl von Braillezeilen umgehen. Bitte installieren Sie BRLTTY für Windows, um Unterstützung für diese zu erhalten: http://brl.thefreecat.org/brltty.

Übersetzungstabelle

Die nächste Option ist die Auswahlliste der Braille-Tabellen, wo die entsprechende Tabelle für die benötigte Sprache mit den Cursortasten ausgewählt werden kann.

Aktuelles Wort ausschreiben

This option allows the word that is under the cursor to be displayed in non-contracted computer braille.

Cursor-Blinkfrequenz

This option is a numerical field that allows you to change the blink rate of the cursor in milliseconds.

Zeitüberschreitung für Meldungen (in Sekunden)

This option is a numerical field that controls how long system messages are displayed on the braille display.

Verschiedenes

When pressing NVDA+Control+T, you can control what is being displayed on the braille display. The two options are focus and review. Focus displays what is being accessed via the arrow keys, the tab key, in short, focus navigation. Review allows you to review what is on the screen, I.E object navigation. You can cycle between these options by pressing control+NVDA+T.

6.2.1.5 Tastatureinstellungen

This dialog box is found in the Preferences menu, under "Keyboard settings...". Alternatively, it can be accessed by pressing NVDA+Control+K. It contains the following options:

Tastaturbelegung

This combo box lets you choose what type of keyboard layout NVDA should use. Currently the two that come with NVDA are Desktop and Laptop.

Dauergroßschreibtaste als NVDA-Taste verwenden

If this checkbox is checked, capslock can be used as an NVDA modifier key.

Einfügetaste der &erweiterten Tastatur als NVDA-Taste verwenden

If this checkbox is checked, the extended insert key (usually found above the arrow keys, near home and end) can be used as an NVDA modifier key.

Einfügetaste des &Nummernblocks als NVDA-Taste verwenden

If this checkbox is checked, the insert key on the number pad can be used as an NVDA modifier key.

Eingegebene Zeichen ansagen

A check box that when checked means that NVDA will announce all characters you type on the keyboard. You can also configure this option from anywhere by pressing NVDA+2.

Eingegebene Wörter ansagen

A check box that when checked means that NVDA will announce all words you type on the keyboard. You can also configure this option from anywhere by pressing NVDA+3.

Funktionstasten ansagen

A check box that when checked means that NVDA will announce all non-character keys you type on the keyboard. This includes key combinations such as control plus another letter. You can also configure this option from anywhere by pressing NVDA+4.

6.2.1.6 Mauseinstellungen

The Mouse Settings dialog is found in the Preferences Menu, under "Mouse settings...". Alternatively, it can be accessed by pressing NVDA+Control+M. It contains the following options:

Form des Mauszeigers ansagen

A checkbox, that when checked means that NVDA will announce the shape of the mouse pointer each time it changes. The mouse pointer in Windows changes shape to convey certain information such as when something is editable, or when something is loading etc.

Text unter dem Mauszeiger ansagen

A check box that when checked means that NVDA will announce the text currently under the mouse pointer, as you move it around the screen. This allows you to find things on the screen, by physically moving the mouse, rather than trying to find them through object navigation.

Objekttyp unter dem Mauszeiger ansagen

If this checkbox is checked, NVDA will announce the role (type) of object as the mouse moves in side it.

Audiokoordinaten bei Mausbewegungen abspielen

Checking this checkbox makes NVDA play beeps as the mouse moves, so that the user can work out where the mouse is in regards to the dimentions of the screen.

Lautstärke der Audiokoordinaten durch Helligkeit kontrollieren

If the 'play audio coordinates when mouse moves' checkbox is checked, then checking this checkbox means that the volume of the audio coordinates beeps is controled by how bright the screen is under the mouse. This may cause some performence issues on Windows Vista, so it is unchecked by default.

6.2.1.7 Einstellungen für Objektdarstellungen

Found in the Preferences menu under "Object Presentation..." or by pressing NVDA+Control+O, this dialog box contains the following options:

Minihilfen ansagen

A check box that when checked tells NVDA to announce tool tips as they appear. Many Windows and controls show a small message (or tool tip) when you move the mouse pointer over them, or sometimes when you move the focus to them.

Hilfesprechblasen ansagen

This check box when checked tells NVDA to announce help balloons as they appear. Help Balloons are like tool tips, but are usually larger in size, and are associated with system events such as a network cable being unplugged, or perhaps to alert you about Windows security issues.

Kurztasten der Objekte ansagen

When this check box is checked, NVDA will include the shortcut key that is associated with a certain object or control when it is announced. For example the File menu on a menu bar may have a shortcut key of alt+f.

Objektbeschreibungen ansagen

Uncheck this checkbox if you feel you don't need to hear the description announced along with objects.

Objektstatus zuerst ansagen

When this check box is checked, NVDA will announce the state of an object before any other information. For example a button with a name of "play" and a state of "pressed" will be announced as "pressed play button" rather than "play button pressed". This is sometimes useful if you are tabbing around a lot of objects of the same type, but you are only really interested to see whether they are "pressed", or "checked" etc.

Signaltöne bei Aktualisierungen des Fortschrittsbalkens

A progress bar is a control which looks a bit like a ruler. As a task is slowly being completed, more of the bar gets highlighted. It also shows a percentage value, to numerically tell you how far along it is. Progress bars are shown for things like loading a web page, checking your email, or processing a sound file etc. If this check box is checked, NVDA will use beeps to let you know how far along a progress bar is as it moves. This is sometimes a lot nicer than hearing lots of numbers.

6.2.1.8 Einstellungen zur virtuellen Ansicht

The Virtual Buffer settings dialog can be found in the Preferences menu, under "Virtual Buffers...". Alternatively, it can be accessed by pressing NVDA+Control+B. Note: Because virtual buffer support is being rewritten for applications such as microsoft internet explorer, outlook, and Adobe Reader, some of the features that are presented in this dialog only apply to the virtual buffers that have already been completed, such as the ones found in Mozilla Firefox 3, Thunderbird 3, and Seamonkey 2.

The dialog contains the following options:

Anzahl Zeichen pro Zeilee

This field sets the maximum length of a line of a virtual buffer (in characters). Its not guaranteed that all lines will end up being this length or under, but most should.

Anzahl Zeilen pro Seitee

Although virtual buffers don't really have pages, this field sets the amount of lines you will move by when pressing page up or page down while in a virtual buffer.

Fokus-Änderung in der virtuellen Ansicht ansagen

This check box when checked tells NVDA to announce the text and presentation information in the virtual buffer for a control, as it gets focus, rather than simply asking Windows for the control information. For example, tabbing to a link called "cats" will say "link cats" rather than "cats link" because in virtual buffers when arrowing around, the type is always said before the text. It may be desirable in certain cases then to not have this option enabled since this means the control will be announced in a some what better order. However, for some web browsers, you may get more accurate information announced when this is enabled.

Ankündigungen von Feldern wie beispielsweise links und Überschriften

Previous versions of NVDA allowed you to use the Virtual Buffer settings dialog to configure whether NVDA should announce the type of field that you entered while reading through web content. Examples of field types are links, headings, tables, and lists. To configure these, please find new options in the Document Formatting Settings dialog. These options now not only affect virtualBuffers, but any documents that may cause these types of fields to be announced.

Bei Fokus-Änderungen automatisch den Fokus-Modus einschalten

This option allows focus mode to be invoked if focus changes. For example, when on a web page, if you press tab and you land on a form, if this option is checked, focus mode will automatically be invoked.

Bei Bewegungen des System-Cursors automatisch den Fokus-Modus einschalten

This option, when checked, allows NVDA to enter and leave focus mode when using arrow keys. For example, if arrowing down a webpage and you land on an edit box, NVDA will automatically bring you into focus mode. If you arrow out of the edit box, NVDA will put you back in browse mode.

6.2.1.9 Dokumentformatierungseinstellungen

This dialog box is found in the Preferences menu, under "Document Formatting...". Alternatively, this dialog box can be invoked by pressing NVDA+Control+D. All the check boxes in this dialog are for configuring what type of formatting you wish to hear automatically as you move the cursor around microsoft word or wordpad documents. For example, if you check the report font name check box, each time you arrow onto text with a different font, the name of the font will be announced.

You can configure announcement of font name, font size, font attributes, style, text alignment, text style, tables, page numbers, line numbers, spelling errors, links, headings, lists and blockquotes.

6.2.2 Speichern und Wiederherstellen der Konfiguration

NVDA stores its settings in a file called nvda.ini in its directory. You will probably never have to touch this file, but for advanced users, this file does allow you to change some extra settings which don't yet have dialog boxes to do so.

NVDA does not automatically save your settings, so if you exit NVDA, and then restart it, the settings will go back to the original values. Note, however, that this default option can be changed under the general options in the preferences menu. To save the settings, you can either choose the Save configuration item in the NVDA menu, or press NVDA+Control+c from anywhere.

If NVDA is on a read-only file system, such as a CD, it will not be able to save your settings, and it will tell you so.

If you ever make a mistake with your settings and need to revert back to the saved settings, you can either choose the "revert to saved configuration" item in the NVDA menu, or you can press NVDA+Control+r from anywhere.

7. In NVDA navigieren

7.1 Navigieren mit dem Fokus

The Most common way of navigating around Windows with NVDA, is to just move around with normal keyboard commands, such as tab and shift tab to move forward and back between controls, pressing alt to get to the menu bar and then using the arrows to navigate menus, using alt-tab to move between running applications. As you do this, NVDA will announce what has focus.

When NVDA announces an object (e.g. when it receives focus), it will usually say the object's name, type, state, value, description, keyboard shortcut, and positional information. This is quite similar to other screen readers; however the types of objects may be a little different.

There are some key commands that are useful when moving with focus:

  • NVDA+tab - reports the current focus
  • NVDA+b - reads the entire foreground window (useful for reading a dialog box)
  • NVDA+t - Announces the title of the current foreground window

7.2 Navigieren mit dem System-Cursor

When the focus is on an object that has an edit cursor (system caret), you can arrow around and edit like you normally would. NVDA will announce as you move by character, word, line, and will also announce selecting and unselecting text.

When you move with the system caret, the position of the review cursor will also be updated to match the system caret.

You can also read the entire text (say all) with NVDA+down Arrow. Note though that this is different to numpad Plus which only moves the review cursor, this moves the system caret as it goes.

7.3 Objektnavigation

Wenn Sie das Betriebssystem erkunden möchten, ohne den Fokus dabei zu benutzen, können Sie die Objektnavigation von NVDA verwenden. Dies ermöglicht Ihnen, sich in einer baumartigen Struktur zwischen den Objekten zu bewegen.

Die Baumstruktur beginnt mit der obersten Eben, dem Desktop. Untergeordnet folgen die einzelnen laufenden Anwendungen. Die Anwendungsfenster können ebenfalls Unterfenster enthalten.

Die Baumstruktur kann bei manchen Anwendungen sehr umfangreich werden. Auf Grund der logischen Anordnung der Objekte, werden Sie die benötigten Informationen dennoch schnell finden können.

Dies repräsentiert weniger die Anordnung der Objekte auf dem Bildschirm als vielmehr die logische Anordnung innerhalb des Programms. Es wird zwar eine Weile dauern, bis Sie sich eingearbeitet haben. Aber nach einer gewissen Zeit werden Sie sich schnell und einfach durch die Objekthierarchie bewegen können.

NVDA bietet im Moment keine Möglichkeit, bildschirmorientiert zwischen den Objekten zu navigieren. Das ist deshalb nicht möglich, weil NVDA sich nicht in die Grafikdarstellung mittels eines InterCept-Treibers hineinhängt. ** #kurze Erklärung in ein oder zwei Sätzen zu InterCept-Treiber fehlt!

Nicht alle Anwendungen haben eine perfekte Baumstruktur; es liegt jedoch am Entwickler jeder einzelnen Anwendung. Die meisten Anwendungen scheinen jedoch eine ordentliche Baumstruktur zu haben.

Um zwischen den Objekten zu navigieren, verwenden Sie die folgenden Tasten:

  • NVDA+NumPad 5 = Aktuelles Objekt ansagen
  • NVDA+NumPad 8 = Zum übergeordneten Objekt springen
  • NVDA+NumPad 4 = Zum vorherigen Objekt springen
  • NVDA+NumPad 6 = Zum nächsten Objekt springen
  • NVDA+NumPad 2 = Zum ersten untergeordneten Objekt springen
  • NVDA+Umschalt+NumPad 4 = Rekursiv zum vorherigen Objekt springen
  • NVDA+Umschalt+NumPad 6 = Rekursiv zum nächsten Objekt springen
  • NVDA+NumPad Minus = Zum Fokus springen
  • NVDA+NumPad Eingabetaste = Aktiviert das aktuelle Objekt
  • NVDA+NumPad Schrägstrich = Zieht die Maus zum aktuellen Objekt
  • NVDA+numPad Stern = Zur Mausposition springen
  • NVDA+Umschalt+NumPad 5 = Ansage der Größe und Position des aktuellen Objekts
  • NVDA+NumPad Plus = Ansage aller rekursiven Objekte vom aktuellen Objekt

Hinweis: Damit NVDA die Tasten vom Nummernblock verwenden kann, muss der Nummernblock ausgeschaltet sein.

Wenn Sie bei der Fokusnavigation nicht möchten, dass der Fokus vom NVDA-Cursor verfolgt wird, betätigen Sie einmal NVDA+7, um ihn abzukoppeln.

7.4 Die Textkontrolle im aktuellen Objekt

Um den Text innerhalb eines Objekts zu erfahren, wie ein bestimmtes Wort geschrieben wurde oder wenn Sie ein Textabschnitt wortweise lesen möchten, können Sie die Tasten des Nummernblocks wiefolgt verwenden:

  • Umschalt+NumPad 7 = Zur ersten Zeile springen
  • NumPad 7 = Vorherige Zeile
  • NumPad 8 = Aktuelle Zeile
  • NumPad 9 = Nächste Zeile
  • Umschalt+NumPad 9 = Zur letzten Zeile springen
  • NumPad 4 = Vorheriges Wort
  • NumPad 5 = Aktuelles Wort
  • NumPad 6 = Nächstes Wort
  • Umschalt+NumPad 1 = Zeilenanfang
  • NumPad 1 = Vorheriges Zeichen
  • NumPad 2 = Aktuelles Zeichen
  • NumPad 3 = Nächstes Zeichen
  • Umschalt+NumPad 3 = Zeilenende
  • NumPad Plus = Gesamten Text ansagen

Hinweis: Damit NVDA die Tasten des Nummernblocks nutzen kann, muss der Nummernblock ausgeschaltet sein.

Normalerweise wird der Text innerhalb eines Objekts aus dessen Name, Wert und Beschreibung zusammengesetzt. Für Objekte, die tatsächlich Text enthalten, wird der in einem Objekt enthaltene Text gelesen. Dies gilt beispielsweise für Eingabefelder und für die Eingabeaufforderung.

Es ist möglich, Textteile zu kopieren, den Sie überprüfen möchten. Beispielsweise, Ihnen wurde von jemandem in eine Chat-Nachricht eine Internet-Adresse zugesendet, die Sie wiefolgt kopieren können: Den Cursor auf das erste Zeichen des zu kopierenden Abschnitts bewegen und NVDA+F9 drücken. Dann mit dem Cursor an das Ende des zu kopierenden Abschnitts wandern, und NVDA-F10 drücken. Damit wird der "markierte" Abschnitt in die Zwischenablage kopiert, um ihn anschließend an den Ort Ihrer Wahl wie gewohnt von Windows aus er Zwischenablage einzufügen. Beachten Sie, dass mit dem zu verwendenden Programm die Navigation mit den Cursor-Tasten möglich ist, um komfortabler mit der "Copy and Paste"-Funktion von Windows zu arbeiten.

7.5 Navigieren mit der Maus

Wenn Sie den Mauszeiger bewegen, liest NVDA standardmäßig den Text, der sich direkt unter dem Mauszeiger befindet, vor. Falls unterstützt, wird der ganze Satz vorgelesen. In einigen Objekten hingegen ist nur das Zeilenweise Lesen möglich.

NVDA kann auch so konfiguriert werden, dass der Name des Objekts bzw. des Steuerelements, wie Listen und Schalter, angesagt werden, wenn sich der Mauszeiger darüber bewegt wird. Dies kann für Blinde insofern nützlich sein, wenn der Text alleine nicht zu erschließen ist.

Weiterhin kann NVDA die Position des Mauszeigers durch Signaltöne akkustisch die Koordinaten wiedergeben. Mit dieser zusätzlichen Funktion können Sie sich mit der Maus am Bildschirm orientieren und somit die aktuelle Position des Mauszeigers ermitteln. Sobald die Maus bewegt wird, werden sogleich die Signaltöne abgespielt. Dies geschieht nach folgenden Regeln: Wenn Sie den Mauszeiger horizontal bewegen, wandert der Signalton entsprechend im Stereofeld, vorausgesetzt Sie verwenden Stereo-Lautsprecher. Wenn Sie den Mauszeiger vertikal bewegen, verändert der Signalton seine Frequenz. Wenn Sie den Mauszeiger nach oben bewegen, wird der Signalton entsprechend höher. Und wenn Sie den Mauszeiger nach unten bewegen, wird der Signalton tiefer. Trifft der Mauszeiger auf helle Bzw. dunkle Bereiche des Bildschirms, verändert der Signalton seine Lautstärke. Trifft der Mauszeiger auf helle Bildschirmbereiche, wird der Signalton lauter. Trifft der Mauszeiger auf dunkle Bildschirmbereiche, wird der Signalton leiser.

Diese Extra-Funktionen sind standardmäßig ausgeschaltet. Eingeschaltet können diese im Optionensmenü.

8. Die virtuellen Ansichten

Complex read-only documents, such as web pages, are represented in NVDA with in a Virtual Buffer.

A virtual buffer is a flat representation of content such as a web page, that can be moved around in with the arrow keys. Information such as whether text is a link, heading etc is announced along with the text as you move.

While in a virtual buffer, there are two modes of interaction. Browse mode is where you can move around all the text using a virtual cursor. Focus mode is where you can directly interact with the control (edit field, combo box, radio buttons) using what ever keys you would normally use. When in Browse mode, NVDA will automatically switch to focus mode if you tab to or click on a particular control that requires it. You can also manually change to focus mode with NVDA+space. If NVDA automatically changed to focus mode because of a tab or click, it will change back to browse mode once you tab or click on to something that doesn't require it. However, if you changed to focus mode with NVDA+space, it will stay in focus mode until you manually go back to browse mode. At any time when you are in focus mode, you can manually switch back to browse mode by pressing escape, or NVDA+space. The virtual buffer settings dialog in the preferences menu in the NVDA menu allows you to disable automatic switching of browse mode and focus mode.

Currently NVDA uses Virtual Buffers for reading documents in Mozilla Firefox, Microsoft Internet Explorer, and Adobe Reader.

Note that the virtualBuffers used for Internet Explorer and Adobe Reader are much more basic, and take much more time to load. These older virtualBuffers also cannot deal very well with dynamic content, nore do they support a Links List or some of the virtualBuffer quick keys. For best performance and ease of use, try Mozilla Firefox for browsing the Web.

8.1 Zeichenweises navigieren

While in browse mode, For quicker navigation NVDA also provides single character keys to jump to certain fields in a virtual buffer.

By themselves they jump to the next field, with the shift key they jump to the previous field.

Tasten, die in den neuen und älteren virtuellen Ansichten unterstützt werden:

  • H = Überschriften
  • L = Liste
  • I = Listeneintrag
  • T = Tabelle
  • K = Link
  • P = Absatz (verwendbar nur in den älteren virtuellen Ansichten, Steuerung+Pfeil nach oben / Steuerung+Pfeil nach unten in den neueren virtuellen Ansichten verwenden)
  • F = Formularfeld

Die Tasten in dem älteren virtuellen Ansicht werden nicht unterstützt:

  • U = nichtbesuchter Link
  • V = Besuchter Link
  • E = Eingabefeld
  • B = Schalter
  • X = Kontrollfeld
  • C = Kombinationsschalter
  • R = Auswahlschalter
  • Q = Zitatblock
  • S = Trennlinie
  • M = Rahmen
  • G = Grafik
  • 1 bis 6 = Jeweilige Überschriften 1 bis 6

8.2 Formular- und Durchreiche-Mode

Manchmal können Sie die Interaktion mit einem Formularmodus mit der sogenannten virtuellen Ansicht gesteuert werden. Für diesen Zweck kann der Fokus-Modus zum Ein- / Ausschalten mit NVDA-Leertaste verwendet werden.

8.3 Die Linkliste

Um eine Liste aller Links im aktuell geladenem Virtuellen Betrachtung angezeigt zu bekommen, betätigen Sie NVDA+F7. Im nun erscheinenden Dialogfeld können Sie den gewünschten Link auswählen, zum Link springen oder den Link aktivieren.
