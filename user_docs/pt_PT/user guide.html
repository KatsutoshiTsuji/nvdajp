1. Introdução

Características Gerais

O feedback fornecido pelo NVDA através de voz sintética permite às pessoas cegas acederem e interagirem com o sistema operativo Windows e diversas aplicações de terceiros.

As características principais são:

  • Permite a sua execução directamente através de dispositivos USB ou de outros media portáteis sem necessitar de instalação
  • Navegar na Web com o Mozilla Firefox 3
  • Instalador falante e fácil de usar
  • Trabalha com email que utilizam o Mozilla Thunderbird 3
  • Suporte precário para o Microsoft Internet Explorer
  • Suporte básico para o Microsoft Outlook Express / Windows Mail
  • Suporte básico para o Microsoft Word e Excel
  • Suporte para aplicações Java acessíveis
  • Suporte precário para o Adobe Reader
  • Suporte precário para o IBM Lotus Symphony
  • Suporte para a Prompt de Comando do Windows e aplicações de consola
  • Anúncio automático do texto por baixo do rato e indicação em áudio opcional da posição deste periférico

1.2. Internacionalização

É importante que as pessoas em qualquer parte do mundo, independentemente do idioma que falem, obtenham o mesmo acesso às tecnologias. O NVDA está traduzido em mais de 20 idiomas incluindo: Português do Brasil, Checo, Finlandês, Francês, Alemão, Húngaro, Italiano, Português, Eslovaco, Espanhol, Sueco, Chinês Tradicional, Vietnamita, Holandês Sul-Africano, Galego, Croata, Japonês, Polaco, Russo, Tailandês e Ucraniano.

1.3. Suporte De Sintetizador De Voz

Para além de fornecer mensagens e interface em diversos idiomas, o NVDA também pode permitir ao utilizador ler o conteúdo em qualquer língua, tendo em conta que tenha um sintetizador de voz que possa falar esse idioma em particular.

O NVDA está incluído com o eSpeak, um sintetizador de voz livre, open-source e multi-idioma. Adicionalmente, este leitor de ecrã pode usar ambos os motores de voz SAPI4 e SAPI5 para permitir a saída de voz, bem como com o sintetizador de voz Audiologic.

1.4. Inovação E Experimentação

O NVDA não se encontra limitado pela necessidade de seguir tendências e demandas do mercado. Os Leitores de Ecrã comerciais têm boas características, mas algumas vezes estão dependentes do que o mercado pretende, em vez das mesmas serem realmente úteis para o utilizador. O NVDA poderá não ser sempre tão estável como outros Leitores de Ecrã o são, mas poderá certamente servir como uma ferramenta experimental e de teste. Pode experimentar facilmente e rapidamente novas ideias e características que talvez não tenham sido vistas noutros Leitores de Ecrã para o Sistema Operativo Windows. Um exemplo disso é a capacidade do NVDA utilizar beeps para indicar ao utilizador que uma Barra de Progresso se está a mover. O beep mais alto e agudo indica o fim da mesma. Esta característica foi adicionada a outros Leitores de Ecrã através de scripts personalizados e similares, mas nunca foram oficialmente adoptados completamente no núcleo de nenhum outro Leitor de Ecrã para o Windows.

1.5. Arquitectura E Implementação

O NVDA é escrito na linguagem de programação Python, é construído com uma arquitectura modular e muito do seu código pode ser adicionado e estendido facilmente para dar suporte a novos programas e/ou controlos no Windows. As App Modules podem ser escritas para se adicionar suporte total para uma aplicação específica, os buffers virtuais podem ser escritos para permitirem ao NVDA mostrar documentos complexos ou outros dados e os Objectos deste leitor de ecrã podem ser escritos para se adicionar suporte a controlos específicos ou janelas. O NVDA tenta tornar constantemente os controlos e elementos de um programa ou Sistema Operativo como acessíveis sempre que lhe seja possível, para que o utilizador possa procurar activamente toda a informação pretendida. Este leitor de ecrã não está interessado com características especiais tal como filtrar e anunciar informação particular que considere que o utilizador pretenda saber.

Para comunicar com o Sistema Operativo e programas, o NVDA usa o conjunto de funções do Windows, Microsoft Active Accessibility (MSAA), IAccessible2, Java Access Bridge e interfaces de programação específicas fornecidas por determinadas aplicações. Este leitor de ecrã não utiliza quaisquer drivers específicas de Interceptor de Vídeo ou display hooks e tentará adquirir sempre que possível o maior número de informação a partir das interfaces de programação específicas, antes de recorrer a outro método.

2. Requerimentos Do Sistema

  • Sistemas Operativos: todas as versões do Windows XP e do Windows Vista (incluindo as versões Server) e pode trabalhar parcialmente no Windows 2000
  • Memória: 256 mb ou mais de RAM
  • Velocidade do processador: 1.0 GHz ou superior

3. Obter O NVDA

Se não tem uma cópia do NVDA, pode fazer o download no website deste software em: http://www.nvda-project.org/

Vá a secção download e procure o link para descarregar a última versão do NVDA.

Actualmente, existem duas formas em que o NVDA é distribuído. Uma é um Instalador que falará através da instalação do NVDA de forma que possa executá-lo em qualquer altura a partir do Menu Iniciar. O outro é um ficheiro zip que contém a versão completa portátil deste leitor de ecrã, que pode executá-lo a partir de qualquer lugar, incluindo os dispositivos USB ou CD.

4. Instalar O NVDA

4.1. O Instalador Do NVDA

Se já tem o instalador do NVDA, pressione simplesmente Enter ou um duplo clique no ficheiro e o mesmo iniciará.

Quando a instalação estiver a ser carregada, ouvirá a música de instalação do NVDA. Após estar carregado, uma cópia temporária deste leitor de ecrã permitirá que prossiga pelas prompts do instalador e instale o NVDA. Note que se estiver a executar outra cópia deste leitor de ecrã no momento em que iniciou o instalador, uma mensagem aparecerá a avisá-lo que a sua cópia anterior do NVDA será encerrada. Quando pressionar Enter nesta mensagem, o instalador concluirá a mesma e iniciará a sua própria cópia. Isto é necessário para que o instalador possa actualizar adequadamente qualquer ficheiro alterado.

Uma vez que tenha completado correctamente a instalação, o instalador iniciará finalmente a nova cópia instalada do NVDA para si.

4.2. O Ficheiro Zip Portátil

Se tem a versão do ficheiro zip do NVDA, então tudo o que tem de fazer é descompactar este ficheiro para qualquer lugar que pretenda no seu disco local ou noutro media portátil.

5. Iniciar Com O NVDA

5.1. Executar O NVDA

Se tem o NVDA instalado com o instalador, então iniciar o NVDA é tão simples como pressionar control+alt+n, ou escolher o NVDA a partir do submenu NVDA que se encontra nos Programas no Menu Iniciar.

Para iniciar a versão do ficheiro zip portátil, vá para a pasta onde descompactou o NVDA e pressione Enter ou um duplo clique no nvda.exe.

Assim que o NVDA arranca, ouvirá primeiramente um conjunto de tons ascendentes (a informá-lo que este está a ser carregado). Então deverá ouvir o leitor de ecrã anunciar "NVDA Iniciado". Dependendo da rapidez do seu computador, ou se está a executar o NVDA através de um dispositivo USB ou de outra media mais lenta, poderá demorar uns momentos para iniciar. Se o NVDA estiver a demorar um tempo maior do que o previsto, o leitor de ecrã dirá "A carregar componentes. Por favor aguarde...

Se não ouvir nada disto, ou escutar um som de erro do Windows, ou um conjunto de sons descendentes, então isto significa que o NVDA tem um erro, e necessitará possivelmente de comunicar o bug aos programadores. Por favor procure no website do NVDA para saber como deve proceder.

Quando o NVDA inicia pela primeira vez, ser-lhe-á apresentada uma caixa de diálogo que lhe fornece alguma informação básica sobre a tecla modificadora NVDA e o menu deste leitor de ecrã. (Por favor veja as secções seguintes sobre estes tópicos). A caixa de diálogo contém também duas caixas de verificação. A primeira permite-lhe controlar se o NVDA utiliza o capslock como uma tecla modificadora NVDA e a segunda permite-lhe configurar se este diálogo de boas vindas aparece sempre que o NVDA é iniciado.

5.2. Acerca Dos Comandos Do Teclado Do NVDA

Muitos dos comandos específicos do teclado do NVDA consistem normalmente no pressionamento da tecla modificadora do NVDA em conjunto com uma ou mais outras teclas. Uma excepção disto é os comandos que dão a ordem para se rever o texto que utilizam somente o teclado numérico.

Por defeito o insert do teclado numérico e o outro insert podem ser utilizados como teclas modificadoras NVDA. Contudo, pode configurar também este leitor de ecrã para que a tecla capslock também se torne numa tecla modificadora NVDA. Quando isto fica configurado desta forma, premir ou manter pressionado o capslock faz com que este proceda como uma tecla modificadora NVDA mas ao premir duas vezes rapidamente faz com que funcione como a tecla capslock normal.

Muitas das teclas de comando são mencionadas ao longo deste guia do utilizador mas uma maneira fácil para explorá-las é activar a ajuda do teclado.

Para activar a ajuda do teclado, pressione NVDA+1. Para a desactivar, prima NVDA+1 novamente. Enquanto estiver na ajuda do teclado, ao pressionar qualquer tecla o leitor de ecrã anunciará o que faz (se tiver alguma acção associada). As teclas não executarão a sua função enquanto estiver no modo de ajuda do teclado, desta forma pode pressionar as teclas que pretender.

6. O Menu Do NVDA

O menu do NVDA permite-lhe controlar as definições deste leitor de ecrã, aceder à ajuda, guardar/restaurar a configuração e sair do NVDA.

6.1. Aceder Ao Menu Do NVDA

Para aceder ao menu do NVDA a partir de qualquer lugar do Windows e que este leitor de ecrã esteja em execução, pressione NVDA+n. Quando o menu se abrir, pode usar as setas direccionais para navegar pelo menu e activar um item com a tecla Enter. O menu Preferências permite-lhe configurar o NVDA ao seu gosto, o menu Ferramentas contém uma ou duas ferramentas úteis tais como o visualizador de registo do NVDA e a consola de Python deste leitor de ecrã para programadores, o menu Ajuda permite-lhe aceder ao guia do utilizador, guia rápido de teclas de atalho, e muito mais. O menu NVDA também contém itens que lhe permitem guardar ou restaurar a sua configuração e sair do NVDA sempre que pretender.

O menu deste leitor de ecrã pode ser acedido de outra forma a partir do ícone NVDA no Tabuleiro do Sistema do Windows. Pode dar um clique direito do rato no ícone ou mover-se para o Tabuleiro do Sistema através do pressionamento da tecla Windows+b, andar até ao ícone do NVDA e pressionar a tecla das aplicações.

6.2. Guardar E Restaurar A Configuração

O NVDA guarda as suas definições num ficheiro denominado nvda.ini na sua própria pasta. Provavelmente, nunca terá que tocar nele mas para os utilizadores avançados, este ficheiro permite-lhes alterar algumas definições extra que não se encontram nas caixas de diálogo.

O NVDA não guarda automaticamente as suas definições. Desta forma, se sair do NVDA e reiniciá-lo, estas voltarão ao ponto anterior com os valores originais. Para guardar as definições, pode escolher em qualquer altura o item Guardar definições no menu do NVDA ou pressionar NVDA+Control+c a partir de qualquer lugar.

Se o NVDA tem um ficheiro num sistema só de leitura, como um CD, não será possível guardar as suas definições e o leitor de ecrã informará.

Se nunca se engana com as suas definições e necessita voltar a trás nas configurações guardadas, pode sempre escolher o item "Restaurar para as definições guardadas no menu do NVDA, ou pode pressionar NVDA+Control+r a partir de qualquer local.

6.3. O Menu Preferências

No menu do NVDA há o submenu Preferências que contém diversos itens que abrem caixas de diálogo para que controle várias definições do NVDA.

Pode controlar as definições gerais, selecção de sintetizador, definições de voz, definições do teclado, definições do rato, apresentação de objecto, definições do buffer virtual e definições de formatação de documentos.

7. Navegar Com O NVDA

7.1. Navegar Com O Foco

A forma mais comum de navegar pelo Windows com o NVDA é simplesmente mover-se com os comandos de teclado normais, tais como tab e shift tab para se mover para a frente e para trás entre controlos, pressionar alt para obter a barra de menus e utilizar as setas para navegar pelos menus e usar alt-tab para se mover entre aplicações em execução. Ao fazer isto, o NVDA anunciará o que está sobre o foco.

Quando o NVDA anuncia um objecto (ex.: quando recebe o foco), dirá normalmente o nome do objecto, tipo, estado, valor, descrição, atalho do teclado e informação da posição. Isto é bastante similar noutros leitores de ecrã, o que não impede que os tipos de objectos possam ser um pouco diferentes.

Existem alguns comandos de teclado úteis quando se move com o foco:

  • NVDA+tab  informa onde está o foco actual
  • NVDA+b  lê toda a janela em primeiro plano (útil para ler uma caixa de diálogo)
  • NVDA+t  Anuncia o título da janela actual em primeiro plano

7.2. Navegação De Objectos

Se pretende ter uma visão do sistema Operativo mas sem mover o foco, pode utilizar a navegação de objectos do NVDA. Isto permite mover-se de um objecto para outro numa árvore estruturada.

A árvore estruturada dos objectos inicia-se na janela do ambiente de trabalho, ou seja, na raiz da mesma. As primeiras ramificações da árvore são as aplicações actualmente em execução, e como é óbvio, estas aplicações têm as suas próprias ramificações (objectos), que representam diversas partes da aplicação.

A estrutura em árvore pode ser muito grande, mas permite-lhe navegar numa ordem lógica, de forma a conseguir encontrar a informação que necessita.

Esta estrutura não tem nada a ver com a forma em que os objectos estão dispostos no ecrã, mas sim como estão relacionados logicamente. Isto poderá demorar algum tempo até que comece a utilizar, mas uma vez que compreenda isto, deverá conseguir mover-se com muita facilidade.

Actualmente o NVDA não pode navegar pelos objectos através do layout baseado no ecrã porque não tem nenhum driver de intercepção de vídeo ou display hooks. O NVDA não escolhe interagir com o ecrã porque deve permitir acima de tudo mais clareza e mais percepção lógica de uma aplicação e do sistema operativo, antes de tentar captar a disposição do layout do ecrã.

Nem todas as aplicações têm uma estrutura em árvore perfeita, mas isto depende do programador dessa aplicação em particular. A maioria está correcta e certamente, grande parte das caixas de diálogo e aplicações muito estruturadas parecem estar bem.

Para navegar através de objectos utilize as teclas seguintes:

  • NVDA+numpad5: Anuncia o objecto actual
  • NVDA+numpad8: Move para o objecto ascendente (Movendo na direcção da raiz da árvore)
  • NVDA+numpad4: Move para o objecto anterior (o objecto imediatamente anterior a este no mesmo nível)
  • NVDA+numpad6: Move para o objecto seguinte (O objecto imediatamente seguinte a este no mesmo nível)
  • NVDA+numpad2: Move para o primeiro objecto descendente (O primeiro ramo que sai deste objecto)
  • NVDA+shift+numpad4: Move para o objecto anterior no fluxo (Atravessa automaticamente fronteiras de ascendentes e descendentes até encontrar o anterior)
  • NVDA+shift+numpad6: Move para o próximo objecto no fluxo (Atravessa automaticamente fronteiras de ascendentes e descendentes até encontrar o próximo)
  • NVDA+numpadMenos: Move para o foco
  • NVDA+numpadEnter: Activar o objecto actual (ex. pressionar Enter / clique / duplo clique)
  • NVDA+numpadBarra: Move o rato para o objecto actual
  • NVDA+numpadAsterisco: Move para o rato
  • NVDA+shift+numpad5: anunciar as dimensões actuais dos objectos referente ao ecrã
  • NVDA+numpadMais: Leitura contínua de todos os objectos  Começa a ler do objecto actual, prosseguindo pelos próximos no fluxo

Nota: As teclas do teclado numérico requerem que o numlock esteja desactivado para que possam trabalhar adequadamente.

Quando se mover com o foco, o objecto de navegação actual altera-se juntamente com ele. Contudo, se pretende bloquear a navegação de objecto para um objecto específico, para que este não se altere com o foco, pode desactivar o foco a Mover o Objecto de navegação com NVDA+7.

7.3. Rever Texto No Objecto Actual

Algumas vezes poderá querer saber como se escreve algo, ou talvez necessite ler qualquer coisa por palavras. Pode rever o texto do objecto actual com o teclado numérico:

  • Shift+numpad7: Primeira linha
  • numpad7: Linha anterior
  • numpad8: Linha actual
  • numpad9: Linha seguinte
  • shift+numpad9: Última linha
  • numpad4: Palavra anterior
  • numpad5: Palavra actual
  • numpad6: Palavra seguinte
  • shift+numpad1: Início da linha
  • numpad1: Caracter anterior
  • numpad2: Caracter actual
  • numpad3: Caracter seguinte
  • shift+numpad3: Fim da linha
  • numpadMais: Leitura contínua de todo o texto

Nota: As teclas do teclado numérico requerem que o numlock esteja desactivado para que possam trabalhar adequadamente.

Para a maioria dos objectos, o texto interno é composto pelo nome, valor e descrição. O que não impede que para os objectos que actualmente contenham texto real ou conteúdo editável que o texto seja o texto actual no objecto.

Por exemplo: um item de lista numa lista terá texto que contenha o próprio nome, valor e descrição. Mas uma área de edição ou uma janela de consola do dos, terá o texto actual que esteja no campo ou janela.

Quando um objecto contém um cursor de edição (cursor do sistema), e ele move-se, o cursor de revisão também é actualizado para essa posição. Se pretender que o cursor de revisão permaneça onde está e não se mova com o cursor do sistema, pode desactivar o Cursor Move Cursor de Revisão através do pressionamento de NVDA+6.

7.4. Navegar Com O Cursor do Sistema

Quando o foco está sobre um objecto que tem um cursor de edição (cursor do sistema), pode andar com as setas e editar normalmente como pretender. O NVDA anunciará como se mover por Caracter, palavra, linha e também informará a selecção e não selecção de texto.

Quando se mover com o cursor do sistema, a posição do cursor de revisão também será actualizada para se juntar ao mesmo.

Pode também ler todo o texto (leitura contínua) com NVDA+seta abaixo. Note que isto é diferente do numpad Mais o qual move somente o cursor de revisão, enquanto o primeiro move o cursor do sistema conforme se vai movimentando.

7.5. Navegar Com O Rato

Quando mover o rato, o NVDA informará por predefinição o texto que se encontra directamente sobre o ponteiro deste periférico, assim que este se mova pelo o texto. Quando suportado, este leitor de ecrã ler-lhe-á o conteúdo de parágrafos de texto, contudo em alguns controlos só poderá ler por linha.

O NVDA também pode ser configurado para anunciar o tipo de controlo ou o objecto onde está actualmente o rato conforme se movimente (ex. lista, botão etc.), o que é muito útil para utilizadores totalmente cegos quando algumas vezes o texto não for suficiente.

O NVDA fornece um modo para que os utilizadores compreendam onde está o rato no que tange às dimensões do ecrã através da reprodução das coordenadas actuais do rato com beeps audíveis. Quanto mais alto estiver o rato no ecrã, mais agudo será o tom dos beeps. Quanto mais à esquerda ou à direita estiver o rato no ecrã, mais parecerá que o som anda à esquerda ou à direita (assumindo-se que o utilizador tem umas colunas stereo).

Estas características adicionais do rato não se encontram activadas por defeito no NVDA, mas se pretender tirar partido das mesmas, pode configurá-las a partir do diálogo das definições do rato, que se encontram no menu Preferências no menu do NVDA.

8. Buffers Virtuais

Os documentos complexos só de leitura, tais como páginas Web, são representados no NVDA num Buffer Virtual .

Um Buffer Virtual é uma representação textual perfeita do documento, que permite ao utilizador andar para ler o texto e em campos que estão no documento. Para além de conseguir mover-se, também pode pressionar Enter ou espaço em links e botões para os activar, premir Enter ou espaço em áreas de edição para editar o conteúdo respectivo, carregar no Enter ou no espaço em caixas de verificação ou em botões de opção para os seleccionar e pressione Enter ou espaço em caixas de combinação para modificar imediatamente a sua selecção.

Assim que se mova para um campo, o tipo de campo (tais como link, título, lista etc.) é anunciado, o que não significa que esta informação não se encontra fisicamente no buffer, ou seja, pode andar normalmente através do texto.

Actualmente o NVDA utiliza os Buffers Virtuais para ler documentos no Mozilla Firefox, Microsoft Internet Explorer e Adobe Reader.

Note que os Buffers virtuais usados para o Internet Explorer e Adobe Reader são muito básicos e requerem muito mais tempo para carregar. Estes Buffers Virtuais antigos nem sequer tratam muito bem conteúdo dinâmico, nem fornecem suporte à Lista de Links ou a algumas teclas de navegação rápida do Buffer Virtual. Para uma melhor performance e facilidade de uso, teste o Mozilla Firefox para navegar na Web.

8.1. Navegação Por Letra

Para uma navegação rápida, o NVDA também fornece teclas de Caracter simples para saltar para determinados campos no buffer virtual.

Somente a letra salta para o próximo campo, acompanhada com a tecla shift salta para o campo anterior.

Teclas suportadas pelos antigos e novos buffers virtuais:

  • h: título
  • l: lista
  • i: item de lista
  • t: tabela
  • k: link
  • p: parágrafo
  • f: campo de formulário

Teclas não suportadas pelo buffer virtual antigo:

  • u: link não visitado
  • v: link visitado
  • e: edição
  • b: botão
  • x: caixa de verificação
  • c: caixa de combinação
  • r: botão de opção
  • q: bloco de citação
  • s: separador
  • m: frame
  • g: gráfico
  • 1 até 6: títulos de nível 1 até 6 respectivamente

    • 8.2. Formulários E O Modo Passar Através Do Buffer Virtual

    Algumas vezes pretenderá interagir com campos de formulário com as teclas deste buffer virtual, ou seja, as mesmas terão que sair para fora. O modo passar através do buffer virtual foi criado para este problema e pode ser activado e desactivado com NVDA+espaço.

    Contudo, quando pressionar geralmente no campo de formulário, o mesmo será activado automaticamente e quando uma nova página estiver a carregar, o modo passar através do buffer virtual será desactivado.

    8.3. A Lista de Links

    Para aceder à lista de links para o buffer virtual actual, pressione NVDA+f7. A partir do diálogo que aparece, pode seleccionar o link no qual está interessado e então, ou move-se para ele, ou activa-o utilizando os botões fornecidos.

    9. Configurar O NVDA

    9.1. Definições Gerais

    A caixa de diálogo das definições gerais encontra-se no menu Preferências. A mesma contém as seguintes opções:

    Idioma

    Uma caixa de combinação que lhe permite seleccionar o idioma da interface do utilizador e mensagens do NVDA. Existem diversos idiomas, contudo, a última escolha presente na lista denomina-se "Windows". Esta escolha informa ao NVDA para utilizar o idioma no qual o Windows se encontra actualmente (nas Opções Regionais E de Idioma no Painel de Controlo). Atente por favor que o NVDA necessita ser reiniciado. Quando escolher o idioma, este leitor de ecrã questionar-lhe-á se pretende reiniciar e se deseja efectuar a alteração na selecção. Pressione OK.

    Guardar Configuração ao Sair

    Uma caixa de verificação que quando está marcada informa ao leitor de ecrã para guardar automaticamente a configuração actual quando sair do NVDA.

    Alertar antes de sair do NVDA

    Uma caixa de verificação que lhe permite definir se o NVDA questiona ou não se pretende sair deste leitor de ecrã.

    Nível de registo

    Uma caixa de combinação que lhe permite seleccionar a quantidade de informação que o NVDA coloca no registo quando está em execução. Geralmente, os utilizadores não necessitariam de alterar isto já que a informação não é muito registada. Contudo, se pretende fornecer informação ao relatório de bugs etc., então será boa ideia ajustar para um nível mais elevado.

    9.2. Selecção de Sintetizador

    O diálogo de Sintetizador, pode ser encontrado no item "Sintetizadores..." no menu Preferências. Permite-lhe seleccionar qual é o sintetizador que o NVDA utilizará para falar.

    O diálogo tem uma simples caixa de combinação que lista todos os sintetizadores disponíveis. Seleccione o sintetizador que pretende utilizar com as setas e por fim pressione ok. Se existir um erro ao carregar o sintetizador, uma caixa de mensagem aparecerá a alertá-lo para este facto e o sintetizador anteriormente carregado será mantido.

    A lista actual de sintetizadores que o NVDA suporta é: eSpeak, SAPI4, SAPI5, Audiologic, Display, e Silence.

    O sintetizador eSpeak está compilado directamente com o NVDA e não depende de outros quaisquer drivers especiais ou livrarias SAPI para ser instalado. Este leitor de ecrã inicia utilizando o eSpeak por predefinição. O sintetizador em questão trabalhará em qualquer sistema em que o NVDA trabalhe e funcionará definitivamente quando executar o NVDA através de uma unidade de memória USB ou CD, em qualquer outro sistema.

    O NVDA tem actualmente dois drivers SAPI4. As pessoas informam resultados diferentes com ambos, desta forma, teste-os e utilize o que funcionar melhor para si.

    O sintetizador Display é útil para programadores com visão que pretendam ler o que é que o NVDA está a anunciar. Desta forma, podem testar o NVDA com as suas aplicações.

    O driver do sintetizador Silence não é útil, a não ser que não pretenda ter qualquer voz todas as vezes que execute o NVDA.

    Por favor, atente que não importa quantas vozes diferentes SAPI4 ou SAPI5 (ou motores) tem instalados no seu sistema, somente os sintetizadores SAPI4 e SAPI5 actuais serão apresentados no diálogo dos sintetizadores. Para seleccionar um dos motores (vozes), seleccione ou o SAPI4 ou o SAPI5 e então, no diálogo das definições de voz, pode escolher a voz pretendida.

    9.3. Definições de Voz

    O diálogo das definições de voz encontra-se no menu Preferências e contém opções que lhe permitem alterar o som da voz. Pode também configurar diversas opções a partir de qualquer local através do pressionamento de NVDA+Control juntamente com uma das setas. As setas esquerda e direita movem-no através de várias opções e as setas acima e abaixo aumentam ou diminuem essa opção respectivamente.

    Esta caixa de diálogo contém as opções seguintes:

    Voz

    Uma caixa de combinação que lista todas as vozes do sintetizador actual. Pode utilizar as setas para andar pelos diversos itens.

    Variante

    Uma caixa de combinação que lhe permite seleccionar a Variante do sintetizador com a qual deveria falar. Actualmente é só o eSpeak que suporta esta definição. As variantes do eSpeak são um pouco como vozes, que proporcionam atributos ligeiramente diferentes da original do mesmo.

    Velocidade

    Uma barra de controlo que lhe permite andar a partir do 0 até ao 100, (sendo 0 a velocidade mais baixa e 100 a mais elevada).

    Entoação

    Uma barra de controlo que lhe permite andar a partir do 0 até ao 100, (sendo o 0 a entoação mais grave e 100 a mais aguda).

    Volume

    Uma barra de controlo que lhe permite andar a partir do 0 até ao 100, (sendo o 0 o volume mais baixo e 100 o mais alto).

    Inflexão

    Uma barra de controlo que lhe permite escolher quanta inflexão (subida e descida na entoação) que o sintetizador usaria para falar.

    Falar toda a pontuação

    Uma caixa de verificação que quando marcada informa ao NVDA para anunciar todos os símbolos de pontuação como palavras (muito útil para testes de leitura). Quando desmarcada, o NVDA deixa a pontuação sem alterá-la, desta forma, o sintetizador lerá frases com a inflexão correcta. Contudo, os símbolos não serão falados em voz alta.

    Anunciar "maiúscula" para maiúsculas

    Uma caixa de verificação que quando marcada informa ao NVDA para anunciar a palavra "maiúscula" antes de qualquer letra maiúscula, quando se anda sobre a mesma ou anuncia-a quando está a ser escrita. Normalmente, o NVDA aumenta ligeiramente a entoação para qualquer letra maiúscula, mas alguns sintetizadores não suportam isto bem, desta forma, esta opção poderá ser utilizada.

    Beep para maiúsculas

    Se esta caixa de verificação está marcada, o NVDA produzirá um pequeno beep cada vez que está a anunciar um Caracter maiúsculo. Tal como a caixa de verificação 'anunciar maiúscula para maiúsculas', esta é útil para sintetizadores que não possam alterar a sua entoação para letras maiúsculas.

    9.4. Definições do Teclado

    Esta caixa de diálogo encontra-se no menu Preferências, no item "Definições do teclado...". Esta contém as opções seguintes:

    Layout do teclado

    Esta caixa de combinação permite-lhe escolher qual é o tipo do layout do teclado que o NVDA utilizará. Actualmente, os dois que vêm com o NVDA são o Desktop (computador de secretária) e o Laptop (computador portátil)..

    Utilizar o capslock como uma tecla modificadora do NVDA

    Se esta caixa de verificação está marcada, o capslock pode ser usado como uma tecla modificadora do NVDA.

    Utilizar insert como uma tecla modificadora do NVDA

    Se esta caixa de verificação está marcada, a tecla insert (normalmente encontra-se acima das setas, perto do home e end) pode ser usada como uma tecla modificadora do NVDA.

    Utilizar insert do teclado numérico como uma tecla modificadora do NVDA

    Se esta caixa de verificação está marcada, a tecla insert do teclado numérico pode ser utilizada como uma tecla modificadora do NVDA.

    Falar Caracteres Introduzidos

    Uma caixa de verificação que quando marcada informa que o NVDA anunciará todos os caracteres que escrever no teclado. Pode também configurar esta opção a partir de qualquer local através do pressionamento de NVDA+2.

    Falar Palavras Introduzidas

    Uma caixa de verificação que quando está marcada informa que o NVDA anunciará todas as palavras que escrever no teclado. Pode também configurar esta opção a partir de qualquer local através do pressionamento de NVDA+3.

    Falar Teclas de Comando

    Uma caixa de verificação que quando marcada informa que o NVDA anunciará todas as teclas que não são caracteres que premir no teclado. Isto inclui combinações de teclas tais como o control mais outra letra. Pode também configurar esta opção a partir de qualquer local através do pressionamento de NVDA+4.

    9.5. Definições do rato

    O diálogo das definições do rato encontra-se no menu Preferências, no item "Definições do rato...". Contém as opções seguintes:

    Informar Alterações No Rato

    Uma caixa de verificação que quando marcada informa que o NVDA anunciará em qualquer altura a alteração da forma do ponteiro do rato. O ponteiro do rato altera a sua forma no Windows para fornecer determinada informação tal como quando algo é editável, ou quando alguma coisa está a carregar, etc.

    Informar Texto Por Baixo do Rato

    Uma caixa de verificação que quando marcada significa que o NVDA anunciará o texto actual sobre o ponteiro do rato, conforme se move pelo ecrã. Isto permite que procure coisas no ecrã, através do movimento físico do rato, antes que trate de encontrá-las através da navegação de objectos.

    Anunciar Tipo De Controlo Quando O Rato Entrar No Objecto

    Se esta caixa de verificação está activada, o NVDA anunciará o tipo do objecto quando o rato se mover sobre ele.

    Tocar coordenadas de áudio quando o rato se mover

    Ao marcar esta caixa de verificação faz com que o NVDA reproduza beeps quando o rato se mover. Desta forma, o utilizador pode saber onde é que o rato está tendo em conta as dimensões do ecrã.

    Brilho controla o volume das coordenadas de áudio

    Se a caixa de verificação 'tocar as coordenadas de áudio quando o rato se mover' está marcada, então ao seleccionar esta caixa de verificação significa que o volume dos beeps das coordenadas de áudio é controlado através do brilho do ecrã que está sobre o rato. Isto poderá causar alguns problemas de rendimento no Windows Vista, desta forma, está desmarcada por defeito.

    9.6. Definições de Apresentação de Objecto

    Encontra-se no menu Preferências no item "Apresentação de Objecto..." esta caixa de diálogo contém as opções seguintes:

    Informar Sugestões

    Uma caixa de verificação que quando marcada informa ao NVDA para anunciar sugestões conforme apareça. Muitas janelas e controlos mostram uma pequena mensagem (ou sugestão) quando move o ponteiro do rato sobre elas, ou algumas vezes quando move o foco por elas.

    Informar Balões de Ajuda

    Esta caixa de verificação quando marcada informa ao NVDA para anunciar balões de ajuda conforme apareça. Os Balões de Ajuda são como pequenas sugestões, mas por norma são maiores em tamanho e estão associados com os eventos do sistema tal como o cabo de rede ser desligado, ou talvez para alertá-lo sobre problemas de segurança do Windows.

    Informar Teclas de atalho de Objectos

    Quando esta caixa de verificação é seleccionada, o NVDA incluirá a tecla de atalho que é associada com um determinado objecto ou controlo quando o mesmo é anunciado. Por exemplo o menu Ficheiro na barra de menus poderá ter a tecla de atalho alt+f.

    Informar Descrições de Objectos

    Desmarque esta caixa de verificação se não necessita ouvir a descrição anunciada juntamente com os objectos.

    Dizer Primeiramente Estado de Objectos

    Quando esta caixa de verificação está marcada, o NVDA anunciará o estado de um objecto antes de qualquer outra informação. Por exemplo, um botão com o nome "reproduzir" e estado "pressionado" será anunciado como "pressionado reproduzir botão" em vez de "reproduzir botão pressionado". Isto é algumas vezes útil se andar com o Tab por objectos do mesmo tipo e está realmente só interessado em ver se estão "pressionados", ou "marcados" etc.

    Dar Beeps em Actualizações de Barras de Progresso

    Uma barra de progresso é um controlo que se parece um pouco com uma régua. Conforme uma tarefa está a ser lentamente completada, a barra vai aumentando. Também mostra um valor de percentagem numérico informando-o quando vai avançando. As barras de progresso são apresentadas para coisas como carregamento de páginas da Web, descarregar o seu e-mail, ou processar um ficheiro de som etc. Se esta caixa de verificação estiver marcada, o NVDA utilizará beeps para o informar sobre o avanço da barra de progresso. Isto é algumas vezes mais agradável do que ouvir muitos de números.

    9.7. Definições do Buffer Virtual

    O diálogo das definições do buffer virtual pode ser encontrado no menu Preferências, no item "Buffers virtuais...". Atente que muitas destas definições neste diálogo poderão afectar somente os buffers virtuais antigos (usados para o Internet Explorer e Adobe Reader). Em versões futuras do NVDA este problema será resolvido.

    O diálogo contém as opções seguintes:

    Número máximo de Caracteres Por Linha

    Este campo define o comprimento máximo de uma linha do buffer virtual (em caracteres). Não garante que todas as linhas acabarão por ter esta largura ou inferior, mas a maioria deveria ser assim.

    Número Máximo de Linhas Por Página

    Pois bem, os buffers virtuais não têm realmente páginas. Este campo define a quantidade de linhas ao mover-se quando pressiona page up ou page down no buffer virtual.

    Informar apresentação virtual em alterações do foco

    Esta caixa de verificação quando seleccionada informa ao NVDA para anunciar o texto e informação sobre apresentação no buffer virtual para um controlo, conforme obtenha o foco, em vez de questionar simplesmente ao Windows pela informação do controlo. Por exemplo, ao andar com o Tab para um link denominado "gatos" dirá "link gatos" em vez de "gatos link" porque nos buffers virtuais quando navegar, o tipo é sempre anunciado antes do texto. Poderá ser conveniente que em certos casos não tenha esta opção disponível já que isto significa que o controlo será anunciado numa ordem um pouco melhor . Contudo, para alguns browsers da Web, poderá obter um anúncio mais exacto quando isto está disponível.

    Actualizar Conteúdo Dinâmico

    Quando esta caixa de verificação está marcada, os buffers virtuais têm que permitir que as alterações de páginas sejam actualizadas. Por exemplo, java script. Actualmente, o NVDA somente suporta actualizações de conteúdos dinâmicos no Mozilla Firefox, mas desejamos que o Internet Explorer também suporte esta característica em versões futuras.

    Definições específicas de Anúncio para campos

    As outras onze caixas de verificação nesta caixa de diálogo permitem-lhe configurar que tipo de informação é anunciada quando navegar no buffer virtual. Por exemplo, informar links, o leitor de ecrã dirá a palavra "link" antes de qualquer texto que é um link quando a caixa de verificação está marcada.

    Pode configurar links, títulos, gráficos, listas, itens de lista, tabelas, formulários, campos de formulário, blocos de citação, frames e parágrafos.

    9.8. Definições de Formatação de Documentos

    Esta caixa de diálogo está no menu Preferências, no item "Formatação de Documentos...". Todas as caixas de verificação neste diálogo permitem-lhe configurar qual é o tipo de formatação que pretende ouvir automaticamente conforme se move com o cursor pelos documentos no WordPad ou no Microsoft Word. Por exemplo, se seleccionar a caixa de verificação para informar o nome do tipo de letra de cada vez que anda pelo texto com tipos de letra diferentes, o nome do tipo de letra será anunciado.

    Pode configurar o nome do tipo de letra, tamanho do tipo de letra, atributos do tipo de letra, alinhamento de texto, estilo de texto, tabelas, números de página e números de linha.
